# 测试计划

## 目的

本文档定义技术 MVP M1、M2 和后续目标架构的测试范围。

## 适用范围

适用于 QA、产品、前端和后续 RAG、LLM、安全、集成团队评审。当前阶段验证本地 Vue 前端、Spring Boot Mock API、OpenAPI 契约和人工降级边界。

## 正文

当前阶段测试包括：后端单元与 API 行为测试、Spotless 格式检查、前端 6 项组件行为测试、ESLint、Prettier、生产构建、版本锁定的 OpenAPI 校验、前后端镜像构建及 Compose 冒烟。API 测试验证健康响应以及预诊 200、校验 400、异常 500 的状态码与关键字段；Compose 验证页面、健康接口、`模拟数据` 预诊，并在后端停止后验证前端仍可访问。组件测试继续验证失败不阻断人工提交。

本地验收结束必须执行 `docker compose down --remove-orphans` 并确认无残留容器。测试数据必须标注 `模拟数据`；错误或信息不足不得阻断人工继续提交，智能建议不得替代人工审核。

后续阶段测试包括接口契约测试、RAG 检索评估、LLM 安全测试、集成测试、降级测试、审计测试和权限测试。

任何后续真实数据测试都必须经过授权、脱敏和安全评审。

# 项目进度

## 目的

本文档用于记录 `Transwarp Service Insight` 的当前进度、已完成事项、进行中事项、待办任务、风险阻塞和下一步计划。

## 适用范围

适用于项目例会、阶段评审、AI Agent 协作交接和后续迭代跟踪。

## 总体进度

当前总体进度：8%。

当前阶段：阶段 0，总项目命名与一阶段 `SLA 预诊助手` 路线确认。

## 已完成

- 明确项目总名为 `Transwarp Service Insight`。
- 明确项目方向为借助 AI 能力提升交付与服务效率。
- 明确一阶段能力名称为 `SLA 预诊助手`。
- 明确一阶段聚焦内部员工提交 SLA 前的预诊断。
- 明确一阶段重点是改善问题提交质量，降低基础类、重复类、开源社区类问题的 SLA 提交数量，并提升有效 SLA 的处理效率。
- 明确当前阶段只交付文档与 Demo 方案。
- 明确不写代码、不自动闭单、不假设真实 AIOps 或真实知识库已接入。
- 明确所有模型输出均为辅助建议，必须包含依据来源、置信度、人工介入建议和待补充信息。
- 明确最终结论、工单提交、复盘确认和知识沉淀均需人工审核。

## 进行中

- 细化一阶段 `SLA 预诊助手` 的 Demo 场景和人工审核边界。
- 梳理模拟售后知识库、模拟 SLA 复盘数据、模拟 Wiki、模拟内部培训素材和模拟使用手册的数据结构。
- 梳理 Agent 职责和任务拆解。

## 待办

- 评审文档结构、业务边界和阶段路线。
- 补充 SLA 提交前预诊字段标准。
- 准备模拟知识库样例。
- 准备模拟 AIOps 指标样例。
- 准备模拟 SLA 复盘数据样例。
- 准备模拟 Wiki、内部培训素材和使用手册样例。
- 设计预诊交互样例。
- 细化复盘审核流程。
- 明确后续是否进入原型设计阶段。

## 风险与阻塞

- 真实 SLA 流程尚未确认，当前字段仅为建议方向。
- 真实 AIOps、监控、ITSM、Wiki、售后知识库和工单系统接入状态未确认。
- Demo 使用模拟数据，不能直接代表生产可用性。
- 如后续使用真实数据，必须先完成脱敏、授权、权限和审计设计。
- 若未明确人工审核责任人，复盘和知识沉淀流程无法闭环。
- 二阶段 `智能工单助手` 仅为路线方向，不能作为生产上线方案描述。

## 下一步计划

1. 组织文档评审，确认 `Transwarp Service Insight` 总项目定位和一阶段 `SLA 预诊助手` 范围。
2. 补充 SLA 提交前预诊字段和复盘字段标准。
3. 准备 Demo 所需模拟知识库、模拟 SLA 复盘数据和模拟 AIOps 数据。
4. 形成预诊流程样例和复盘样例。
5. 根据评审结果决定是否进入原型设计阶段。

# Demo 演示话术

## 目的

本文档提供 `SLA 智能预诊助手` 第一阶段纯前端 Demo 的演示话术，帮助演示人员稳定说明项目背景、痛点、交互流程、结果含义和人工审核边界。

## 适用范围

适用于内部演示、方案评审、阶段汇报和 AI Agent 交接。本文涉及的页面字段、参考资料和结果均为 `模拟数据`。

## 正文

### 1. 项目背景

`Transwarp Service Insight` 的目标是借助 AI 能力提升交付与服务效率。当前第一阶段聚焦 `SLA 智能预诊助手`，先在 SLA 提交前增加一个轻量级辅助入口。

### 2. 当前痛点

在 SLA 报备场景中，提交信息可能不完整，处理人需要反复追问版本、组件、复现步骤、日志、影响范围和近期变更。部分基础类、重复类或已有资料覆盖的问题，也可能在提交前缺少统一自查入口。

### 3. Demo 入口

当前页面模拟 AIOps SLA 报备页面。页面中的客户、项目、问题描述、资料链接和结果均为 `模拟数据`，不代表真实客户现场或真实系统数据。

### 4. 点击智能预诊

在原有 `取消` 和 `确定` 按钮之间，我们新增了 `智能预诊` 按钮。这个按钮用于提交前辅助，不替代正式 SLA 提交流程。

### 5. 展示 Loading

点击后，按钮会变为 `预诊中...`，页面展示：

```text
正在检索 Wiki、历史问题、历史 SLA、操作手册和产品文档，请稍候...
```

这里使用前端 `setTimeout` 模拟 1.5 到 3 秒耗时，不调用任何真实接口。

### 6. 展示智能预诊结果

结果面板展示 `智能预诊结果`，包括检索关键词、检索范围、前置诊断结论、建议补充信息、推荐参考资料和操作提示。

需要强调：结果仅基于前端模拟知识库数据，不代表最终根因或处理结论。

### 7. 解释推荐参考资料

推荐资料包括模拟 Wiki、历史问题、历史 SLA、操作手册和产品文档。每条资料都标注为 `模拟数据`，链接使用 `example.com` 示例域，不连接真实内部系统。

### 8. 解释为什么不能自动处理

这个 Demo 的目标不是自动解决 SLA，而是在 SLA 提交前增加一个轻量的智能预诊入口。系统会基于当前表单信息，模拟检索 Wiki、历史问题、历史 SLA、操作手册和产品文档，给用户提供参考资料和补充信息建议。

最终是否提交 SLA、问题分类是否正确、处理建议是否适用于当前环境，都必须由人工确认。当前 Demo 不自动判责、不自动闭单、不自动变更生产配置。

### 9. 说明仍可继续提交 SLA

如果这些资料不能解决当前问题，用户仍然可以继续提交 SLA，由研发或相关责任人进一步处理。`继续提交 SLA` 只关闭结果面板并提示人工确认，不会自动提交。

### 10. 后续演进

后续可以在完成权限、脱敏、审计和人工审核机制后，逐步演进到真实知识库检索、RAG 与大模型总结、AIOps 辅助分析、SLA 表单智能补全和智能复盘报告生成。

这些能力目前只是后续方向，不属于第一阶段 Demo。

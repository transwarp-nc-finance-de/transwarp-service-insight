# 幻觉拦截

## 目的

本文档定义后续 LLM 预诊中的幻觉风险控制策略，降低模型输出不可靠结论的风险。

## 适用范围

适用于大模型安全评审、RAG 质量评估和人工审核流程设计。当前 Demo 不执行真实幻觉检测。

## 正文

生成前拦截：当检索分数低、引用资料不足或关键字段缺失时，不生成诊断建议，只提示补充信息。

生成中约束：Prompt 明确要求只基于引用块回答，不确定时说明需要更多信息。

生成后检测：检查引用对齐、实体一致性、版本号、错误码、产品名和敏感信息。未被引用支持的声明应标记为潜在幻觉，高风险结果降级为仅展示检索列表。

人工抽检：低置信度、高风险、安全拦截和用户负反馈结果应进入人工抽检或知识复审流程。

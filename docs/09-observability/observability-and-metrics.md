# 观测与指标设计

## 目的

本文档定义后续目标架构中的埋点、指标和反馈闭环设计。

## 适用范围

适用于产品效果评估、知识复审、检索质量优化和模型安全评审。当前 Demo 不上报真实埋点。

## 正文

指标包括预诊触发量、预诊结果量、用户采纳率、忽略率、继续提交率、检索命中率、幻觉拦截率、降级次数、平均延迟、知识复审数量、高频未命中 query 和 SME 审核效率。

指标用于衡量服务效率和知识质量，不用于自动判责。任何涉及真实用户或工单的数据都必须脱敏并受权限控制。

### 来源策略观测增补

后续审计事件建议记录 `policy_version`、配置来源、有效来源、实际命中来源、`answer_mode`、降级原因、检索耗时、引用数量和人工确认动作。附件仅记录类型、数量、脱敏状态与不可逆摘要标识，不记录未脱敏正文。

看板可按来源展示启用率、命中率、零命中率、被权限过滤率、通用知识降级率、信息不足率、引用采纳率和继续提交 SLA 比例。指标只用于知识治理与服务效率改进，不用于自动判责、自动闭单或绕过人工审核。

当前 Demo 不上报事件，浏览器 `localStorage` 仅用于模拟配置持久化。

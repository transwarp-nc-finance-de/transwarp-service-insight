# 验收标准

## 目的

本文档定义 Demo、文档、接口草案、RAG、LLM、集成、安全和测试设计的阶段性验收标准。

## 适用范围

适用于 `Transwarp Service Insight` 的阶段评审和后续任务拆分。当前阶段只验收 Demo、PRD、架构文档和 mock 数据，不验收真实生产功能。

## 正文

### 当前阶段验收

- 纯前端 Demo 可本地打开，无需后端服务。
- 所有 Demo 数据和示例均标注 `模拟数据`。
- 智能预诊结果包含依据来源、置信度、人工介入建议和待补充信息。
- 文档明确不接真实 RAG、LLM、AIOps、ITSM、Wiki、历史 SLA 或数据库。
- 不出现真实接口地址、真实客户、真实 SLA、真实日志或真实内部链接。

### 后续设计验收

- API 草案只描述字段、状态和边界，不提供真实服务地址或鉴权实现。
- RAG 设计明确仅 `APPROVED` 知识可进入线上检索。
- LLM 设计明确不输出最终根因或正式处理结论。
- ITSM / AIOps 集成设计明确 AI 不可用时不阻塞人工提交。
- 安全设计覆盖脱敏、权限、提示注入防御、幻觉拦截和审计。
- 测试计划覆盖文档完整性、链接、Demo、RAG 评估、LLM 安全和集成降级。

### 不验收事项

- 不验收真实知识库命中率。
- 不验收真实模型生成质量。
- 不验收真实工单系统联调。
- 不验收真实数据库、真实接口或生产部署能力。

Transwarp Service Insight / 一期产品需求

SLA 预诊助手

面向内部员工提交 SLA 前的智能预诊能力，通过受控知识源、大模型辅助建议，并预留与既有 AIOps 的历史 SLA 查询和 SLA 提交对接要求，在提交源头提升 SLA 信息质量和处理效率。

边界声明：本文为一期 PRD 评审页和下一阶段决策输入，不代表试点已获批，也不代表已完成真实 AIOps、真实 RAG、真实大模型、真实 Wiki、真实历史 SLA 或生产系统接入。当前仍处于第一阶段纯前端 Demo，所有 AI 输出均需人工审核。

重点摘要

产品定位内部员工提交 SLA 前的智能预诊助手，输出建议但不阻断提交。
一期形态可试点系统需求定义，聚焦受控知识源、AI 辅助建议和人工确认流程。
核心指标优先衡量提交质量提升，SLA 数量下降仅作为间接收益。

问题定义

当前 SLA 提交前存在信息不完整、基础类问题重复提交、历史经验未复用、开源社区类问题缺少资料参考、分类判断困难等问题。这些问题会拉长首次理解、首次分派和首次响应耗时。

信息不完整类缺少组件、版本、影响范围、日志、截图、复现条件。
基础类和重复类已有 Wiki、手册或历史摘要可参考，但提交前难以快速命中。
分类不清类提交人难以判断是否继续提交 SLA，以及应先补充哪些信息。

产品目标

  • 提升 SLA 字段完整度、问题上下文、影响范围和附件信息质量。
  • 降低基础类、重复类和资料可自助解决问题的低价值提交。
  • 提高有效 SLA 处理效率，减少首次理解、首次分派和首次响应阶段的低效沟通。
  • 建立可追溯的 AI 辅助输出格式，确保所有建议可解释、可确认、可审计。
  • 为既有 AIOps 的历史 SLA 查询和 SLA 提交能力预留产品接口要求。

用户角色

角色主要诉求一期价值
内部交付、售后、服务员工提交前快速获得参考资料和补充建议。提升提交质量，降低检索成本。
SLA 处理人收到更完整的问题上下文。减少追问，提高分派和响应效率。
知识库维护人员识别高频问题和资料缺口。优化知识库覆盖和质量。
管理和运营人员观察提交质量和效率改进趋势。形成服务效率改进指标。

一期功能范围

SLA 提交前预诊入口在员工填写标题、描述、组件、版本和影响范围后触发。
检索意图生成基于标题、描述、组件、版本、问题类型、错误摘要生成检索意图。
受控知识源检索优先面向售后知识库、Wiki、培训素材、使用手册和处理手册。
AIOps 历史 SLA 查询预留按关键词、组件、版本、类型、时间范围和状态查询脱敏摘要。
AIOps SLA 提交预留员工人工确认后提交标题、描述、附件和预诊摘要。
大模型辅助建议生成摘要、可能原因方向、补充信息和是否继续提交建议。

预诊输出

输出项要求
预诊摘要概括当前问题和已识别上下文。
相似资料展示知识库、Wiki、培训素材、手册或脱敏历史 SLA 摘要。
可能原因方向只能表达为待核验方向，不得表达为最终根因。
建议补充信息列出字段、日志、截图、时间窗口、影响范围或复现步骤。
是否建议继续提交 SLA给出建议但不阻断，员工仍可人工确认后继续提交。
依据来源列出命中的资料来源、历史摘要或输入字段依据。
置信度使用高、中、低表达，并说明影响原因。
人工介入建议明确提交人、处理人或知识库维护人员需要确认的事项。
待补充信息列出进入正式 SLA 前仍缺失的信息。
模拟数据示例：Foundation 审计日志归档功能不可用。依据来源为模拟 Wiki、模拟处理手册和模拟脱敏历史 SLA 摘要；置信度为中；仍需人工补充操作用户、后台日志、归档路径、存储空间和近期变更记录。

AIOps 对接产品接口要求

公司已有 AIOps 系统支持历史 SLA 查询和 SLA 提交。本文只定义后续对接的产品接口要求，不写具体 API 协议、接口地址、鉴权实现或后端代码。

能力产品要求失败处理
历史 SLA 查询按关键词、组件、版本、问题类型、时间范围、处理状态查询授权范围内的脱敏摘要。查询失败不阻断人工提交，提示稍后重试或继续提交。
SLA 提交员工人工确认后提交标题、描述、组件、版本、级别、影响范围、附件信息和预诊摘要。提交失败保留草稿，提示重试或人工处理。
权限与审计按员工身份、角色和可访问范围控制，并记录查询条件、命中来源、预诊结果、提交人和提交动作。审计记录用于追溯和评估，不用于自动判责。

业务流程

员工填写 SLA 标题、描述、组件、版本、级别和影响范围。
员工点击 `SLA 预诊`，系统生成检索意图。
系统检索受控知识源，并预留脱敏历史 SLA 查询。
系统生成预诊建议，展示依据来源、置信度和待补充信息。
员工补充信息、自助处理或人工确认后继续提交 SLA。

指标设计

指标定义价值
SLA 字段完整度提升提交时标题、描述、组件、版本、影响范围、附件等字段完整度变化。衡量提交质量改善。
预诊后补充信息比例员工在预诊后补充字段、日志、截图或影响范围的比例。衡量建议是否推动信息完善。
命中资料占比预诊命中知识资料或脱敏历史 SLA 摘要的占比。衡量知识覆盖和检索效果。
有效 SLA 占比提升提交后无需大量追问、可进入处理流程的 SLA 占比变化。衡量整体提交质量。
首次分派、首次响应、问题理解耗时下降提交到分派、响应和理解问题的平均耗时变化。衡量处理效率提升。

边界风险

一期允许定义
  • 提交前预诊入口。
  • 受控知识源检索能力要求。
  • 大模型辅助建议输出格式。
  • AIOps 历史查询和 SLA 提交的产品预留要求。
  • 权限、脱敏、审计和失败处理要求。
一期明确不做
  • 不自动阻断、自动提交或自动闭单。
  • 不输出最终根因或最终处理结论。
  • 不自动判责或修改生产配置。
  • 不开放给客户或合作伙伴。
  • 不编写真实接口、后端代码或数据库设计。

验收与待确认

验收标准

  • PRD 明确一期背景、目标、角色、范围、非目标、指标和风险边界。
  • 预诊输出包含依据来源、置信度、人工介入建议和待补充信息。
  • AIOps 仅定义产品接口要求，不出现真实接口地址、协议细节或后端实现。
  • 提交流程坚持建议但不阻断，员工可人工确认后继续提交 SLA。

待确认事项

  • 一期试点覆盖团队、组件和问题类型。
  • 知识源准入标准和脱敏审计要求。
  • AIOps 对接排期是否进入一期试点范围。
  • 字段完整度、首次分派、首次响应和有效 SLA 占比的当前基线。
  • 提交人、处理人、知识库维护人员和运营人员的人工审核边界。

# 项目进度

## 目的

本文档记录 `Transwarp Service Insight` 的当前进度、已完成事项、进行中事项、待办任务、风险边界和下一步计划。

## 适用范围

适用于项目例会、阶段评审、AI Agent 协作交接和后续迭代跟踪。当前记录仅覆盖文档、纯前端 Demo、mock 数据、架构设计和演示说明，不代表真实系统接入。

## 正文

### 总体进度

当前总体进度：第一阶段纯前端 Demo 已于 2026/07/10 完成人工授权确认并关闭；架构文档基线进度保持 40%。

阶段状态：`第一阶段：纯前端 SLA 智能预诊 Demo` 已关闭。当前仅准备下一阶段决策材料，尚未进入一期可试点系统设计评审、真实系统开发或真实接入。

### 已完成

(1) 明确项目定位为 `SLA 智能预诊助手`。

(2) 明确第一阶段以纯前端 Demo 为主。

(3) 明确不接入真实后端、真实 RAG、真实大模型或真实 AIOps。

(4) 完成 SLA 页面 `智能预诊` 按钮的原型设计。

(5) 完成模拟知识数据展示方案。

(6) 明确所有 Demo 数据均为 `模拟数据`，不代表真实客户、真实 SLA 或真实知识库。

(7) 明确智能预诊结果不代表最终根因或最终处理结论。

(8) 完成一期 `SLA 预诊助手` PRD 评审稿，包含 Markdown 主文档和 HTML 静态评审展示页。

(9) 在 PRD 中补充 AIOps 历史 SLA 查询和 SLA 提交能力的产品预留要求，明确不包含真实接口协议、后端实现或真实系统接入。

(10) 完成 `docs/` 分层目录重组、架构设计 docx 整合、实施计划、任务拆分、API 草案、RAG、LLM、知识库、集成、安全、观测和测试设计文档。

(11) 完成第一阶段纯前端 Demo 评审定稿，包含 Demo 入口、loading、结果面板、继续提交提示、模拟提交提示和验收说明。

(12) 完成第一阶段 Demo 对话式预诊改造，将原结果抽屉升级为聊天流，支持自动表单摘要、模拟资料卡、快捷反馈、自由输入和前端静态关键词分支。

(13) 完成 Chromium 浏览器技术走查及资料命中复验，覆盖默认 3 条、部分命中 1–2 条、无命中、loading、4 类快捷补充、自由输入低置信度、继续提交、确定和取消；未发现严重问题。

(14) 将模拟参考资料收敛为 Wiki、历史 SLA、产品手册三类候选来源，支持按组件、版本、问题概述和问题描述的本地关键词规则展示 0–3 条；无命中时返回低置信度和人工补充建议。

(15) 完成目标办公环境 Edge 人工冒烟、Foundation 主场景与三类模拟参考资料口径确认，并由授权项目负责人杜恒宇于 2026/07/10 确认第一阶段关闭；未决问题为“无”。

### 进行中

(1) 收集试点用户、知识来源、人工审核责任、数据授权和 AIOps 是否仅预留的决策输入。

(2) 由业务负责人使用获批且脱敏的数据计算并批准字段完整度、有效 SLA 占比、首次分派耗时和首次响应耗时基线。

### 待办

(1) 按 `next-stage-decision-gate.md` 召开阶段决策评审，记录结论、未决事项、负责人和日期。

(2) 仅在全部硬门禁通过后，评审 API 草案、知识权限、安全护栏、审计字段和降级策略。

(3) 暂不开发 RAG、后端，不接入真实大模型、AIOps、Wiki、历史 SLA 或数据库。

### 风险与阻塞

- 当前 Demo 不验证真实检索效果。
- 当前 Demo 的多轮回复来自前端静态关键词分支，不代表真实模型输出。
- 当前 Demo 不包含真实权限、审计、脱敏和接口异常处理。
- 架构文档描述的是目标架构和后续设计草案，不代表当前已实现。
- 后续若接入真实知识库、AIOps 或 SLA 系统，必须单独评审并保留人工审核机制。
- 二阶段 `智能工单助手` 仅为路线方向，不能作为生产上线方案描述。

### 下一步计划

1. 由各责任角色补齐试点团队、用户人数、组件版本、问题类型、排除范围、周期和中止条件。
2. 完成知识来源白名单、授权证明、脱敏规则、访问权限、更新责任、保留周期和审计要求确认。
3. 确认人工审核 RACI，并使用获批且脱敏的数据建立和批准四项核心指标基线。
4. 默认保持 AIOps 仅作能力预留；只有全部门禁通过，才进入一期可试点系统设计评审，且设计评审不等于真实实施授权。

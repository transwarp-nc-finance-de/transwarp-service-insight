# Demo 演示话术

## 目的

本文档提供 `SLA 智能预诊助手` 第一阶段纯前端 Demo 的演示话术，帮助演示人员稳定说明项目背景、对话式交互流程、模拟数据边界和人工审核要求。

## 适用范围

适用于内部演示、方案评审、阶段汇报和 AI Agent 交接。本文涉及的页面字段、参考资料、对话回复和结论均为 `模拟数据`。

## 正文

### 1. 项目背景

`Transwarp Service Insight` 的目标是借助 AI 能力提升交付与服务效率。当前第一阶段聚焦 `SLA 智能预诊助手`，先在 SLA 提交前增加一个轻量级辅助入口，帮助提交人补全信息、查看模拟参考资料，并明确哪些内容仍需人工确认。

### 2. 当前痛点

在 SLA 报备场景中，提交信息可能不完整，处理人需要反复追问版本、组件、复现步骤、日志、影响范围和近期变更。这个 Demo 用对话式预诊方式演示：系统如何在提交前提示用户补充关键上下文，从而提升工单质量。

### 3. Demo 入口

当前页面模拟 AIOps SLA 报备页面。页面中的客户、项目、集群、问题描述、资料链接和对话回复均为 `模拟数据`，不代表真实客户现场或真实系统数据。

底部按钮顺序为：`取消`、`智能预诊`、`确定`。`智能预诊` 是提交前辅助入口，不替代正式 SLA 提交流程。

### 4. 点击智能预诊

点击 `智能预诊` 后，按钮变为 `预诊中...`，抽屉打开并展示 loading。这里使用前端 `setTimeout` 模拟 1.5 到 3 秒耗时，不调用任何真实接口。

建议话术：

```text
这里演示的是第一阶段纯前端 Demo。系统没有接入真实 RAG、大模型、AIOps、Wiki 或历史 SLA，只用本地模拟数据展示交互流程。
```

### 5. 展示对话式智能预诊

loading 结束后，抽屉标题为 `对话式智能预诊`。第一条用户消息由当前 SLA 表单自动生成摘要，表示系统把提交前信息整理成可诊断上下文。

助手第一条回复说明：已基于表单字段和模拟资料完成初步预诊。回复包含诊断结论、依据来源、置信度、人工介入建议和待补充信息。

建议话术：

```text
这里的结论只是辅助建议，不是最终根因，也不是最终处理方案。它的作用是提示提交人：当前还缺少日志、权限、影响范围和近期变更等信息。
```

### 6. 解释推荐参考资料

候选资料收敛为 Wiki、历史 SLA 和产品手册三类，每类最多一条。Demo 根据组件、版本、问题概述和问题描述使用前端本地关键词规则，按实际命中展示 0–3 条模拟资料，不为凑数量展示无关卡片。每条资料都标注 `模拟数据`，链接使用 `example.com` 示例域，不连接真实内部系统。

建议话术：

```text
这些卡片只是前端关键词规则的模拟命中，不代表真实检索结果。部分命中时只显示相关卡片；无命中时会降低置信度并提示补充信息或继续提交 SLA，但不代表问题没有解决方案。真实知识库检索、权限控制、脱敏和审计需要后续单独评审。
```

### 7. 演示多轮反馈

抽屉底部提供 4 个快捷反馈按钮：

- `补充日志现象`
- `补充权限信息`
- `补充影响范围`
- `补充近期变更`

点击任一按钮后，系统追加一条用户消息，并根据前端静态关键词规则返回下一轮模拟回复。

演示顺序建议：

1. 点击 `补充日志现象`，说明系统会提示继续补充日志时间点和组件日志路径。
2. 点击 `补充权限信息`，说明系统会提示核查审计管理权限、角色配置和功能开关。
3. 点击 `补充影响范围`，说明系统会提示确认全员、单用户或单集群影响，并由人工判断 SLA 级别。
4. 点击 `补充近期变更`，说明系统会提示核查升级、配置调整和变更记录。

### 8. 演示低置信度追问

在输入框中输入一段无关内容，例如：

```text
我再看一下页面显示
```

系统会返回“信息仍不足”的模拟回复，并要求按日志、权限、影响范围、近期变更四类继续补充。

建议话术：

```text
当信息不足时，Demo 不会编造最终结论，而是降低置信度并追问关键上下文。这是为了避免把辅助建议误写成正式处理结论。
```

### 9. 继续提交 SLA

如果参考资料和对话追问仍不能解决当前问题，用户可以点击 `继续提交 SLA`。该按钮只关闭抽屉并提示人工确认，不会自动提交、自动闭单或自动处理 SLA。

### 10. 原确定按钮

原页面 `确定` 按钮仍可点击，但当前 Demo 只显示模拟提交提示。实际提交、问题分类、处理建议和复盘结论仍需人工审核。

### 11. 后续演进说明

后续可在完成脱敏、授权、权限、审计和人工审核机制后，再评估真实知识库检索、RAG、大模型总结、AIOps 辅助分析、SLA 表单智能补全和复盘报告草稿生成。这些能力不属于第一阶段 Demo。

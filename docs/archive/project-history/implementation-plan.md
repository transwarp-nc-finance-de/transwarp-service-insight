# 实施计划

## 目的

本文档说明 `Transwarp Service Insight` 如何从当前纯前端 SLA 智能预诊 Demo 演进到可评审、可试点的系统设计方案。

## 适用范围

适用于项目路线评审、阶段拆解、资源评估和后续任务安排。本文仅描述实施计划和目标架构，不代表当前 Demo 已经实现真实 RAG、LLM、AIOps、ITSM 或数据库接入。

## 正文

### 第 0 阶段：仓库清理与文档基线

目标：清理误提交目录，重组 `docs/`，整合架构设计方案，补齐实施计划、任务拆分、API 草案、RAG、LLM、安全、观测和测试设计文档。

交付物：分层文档目录、架构基线 Markdown、提示词归档、更新后的 README、AGENTS 和 CHANGELOG。

边界：不开发后端代码，不接真实系统，不使用真实客户数据。

### 第 1 阶段：纯前端 Demo 增强

目标：优化 SLA 智能预诊入口、loading、结果面板、mock 数据结构、人工审核提示和演示脚本。

交付物：纯前端静态 Demo、mock 数据说明、Demo 验收记录。

边界：所有示例均为 `模拟数据`；不接真实 ITSM、AIOps、RAG、LLM、Wiki 或历史 SLA。

### 第 2 阶段：一期可试点系统设计

目标：评审预诊服务、检索服务、反馈埋点、知识审核、权限脱敏、审计和 ITSM / AIOps 字段对接方案。

交付物：接口草案、数据字段草案、权限与审计设计、集成评审结论。

边界：只做设计和评审，不写真实接口地址、鉴权实现、数据库 Schema 或后端代码。

### 第 3 阶段：受控知识库与检索原型

目标：在授权、脱敏、可审计的数据条件下验证文档解析、切片、元数据标注、Embedding、混合检索、重排序、缓存和检索评估。

交付物：RAG 原型评审报告、检索质量指标、知识缺口列表。

边界：只允许使用授权、脱敏、可审计数据；检索结果必须展示来源和适用范围，不输出最终结论。

### 第 4 阶段：RAG 与大模型预诊

目标：设计 Prompt 模板、结构化输出、引用来源、置信度、待补充信息、幻觉拦截、敏感信息过滤、提示注入防御和降级策略。

交付物：LLM 评审报告、安全测试报告、人工审核流程。

边界：模型输出只作为辅助建议，不作为最终根因、最终处理结论或正式复盘结论。

### 第 5 阶段：ITSM / AIOps 集成与反馈闭环

目标：在员工人工确认后探索工单创建流程嵌入、预诊结果保存、采纳状态记录、指标看板、知识复审和闭环评估。

交付物：集成方案评审、埋点指标设计、降级容灾评审。

边界：AI 能力不可用时不能阻塞工单提交；所有提交动作必须由人工确认。

# 模块边界

## 目的

本文档定义目标架构中各模块的职责边界，避免将辅助预诊能力扩展为自动处理、自动提交或自动闭单能力。

## 适用范围

适用于模块设计、任务拆分和接口评审。当前仓库不实现这些模块，仅保留设计边界。

## 正文

| 模块 | 职责 | 禁止越界 |
| --- | --- | --- |
| 前端预诊面板 | 展示入口、loading、结果、人工审核提示 | 不自动提交 SLA |
| 预诊编排服务 | 组织查询、检索、生成、降级和审计 | 不输出最终根因 |
| 查询改写模块 | 纠错、扩展、规范化、意图识别 | 不覆盖原始输入 |
| 检索服务 | 从审核通过知识中召回资料 | 不检索未授权或未审核知识 |
| LLM 生成模块 | 基于引用资料生成辅助建议 | 不编造来源，不给正式结论 |
| 知识管理模块 | 接入、切片、审核、发布、过期 | 不发布未审核资料 |
| 集成模块 | 与 ITSM / AIOps 交换人工确认后的字段 | 不绕过用户确认 |
| 观测模块 | 记录埋点和降级事件 | 不用于自动判责 |
| 安全模块 | 权限、脱敏、审计和防护 | 不输出未脱敏敏感信息 |

所有模型输出必须包含依据来源、置信度、人工介入建议和待补充信息。

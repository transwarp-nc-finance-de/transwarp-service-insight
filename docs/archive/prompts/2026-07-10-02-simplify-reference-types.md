# 模拟参考资料类型收敛与按命中展示提示词

## 使用时间

2026/07/10

## 使用目的

用于将 `SLA 智能预诊助手` 第一阶段纯前端 Demo 的模拟参考资料从固定四条收敛为三类候选来源，并改为按前端本地关键词实际命中展示，避免为凑数量呈现无关资料。

## 适用范围

适用于 `prototypes/sla-precheck-demo.html` 的静态 mock 数据与交互、Mock 知识说明、Demo 计划与话术、第一阶段验收和关闭记录。不适用于后端代码、真实接口、真实 RAG、真实大模型、真实 AIOps、真实 Wiki、真实历史 SLA、数据库或生产系统接入。

## 提示词正文

### 总结

将参考资料统一为 Wiki、历史 SLA、产品手册三类，删除与历史 SLA 定位重复的“历史问题”，并将“产品文档”统一命名为“产品手册”。每类当前最多维护一条模拟资料，合计按命中展示 0–3 条。

### 关键要求

1. 只基于产品组件、产品版本、问题概述和问题描述四个字段执行前端本地确定性关键词过滤，不调用真实接口或模型。
2. 默认 Foundation 归档场景展示三类模拟资料；部分命中只展示实际相关的 1–2 条；不为凑数量展示无关资料。
3. 无命中时不渲染资料标题、卡片容器或占位卡片，只显示“暂未检索到相关模拟资料，可补充信息或继续提交 SLA”。
4. 依据来源动态说明命中数量；无命中时置信度为低。
5. 无论是否命中，都保留模拟数据边界、依据来源、置信度、人工介入建议、待补充信息和非最终结论声明。
6. “没有相关资料”不等于问题没有解决方案，也不阻止人工继续提交 SLA。
7. 保留日志、权限、影响范围、近期变更四类快捷补充和自由输入分支；它们不是参考资料类型。
8. 使用 Chromium 复验默认、部分命中、无命中和现有交互，并将 Edge 冒烟、业务口径确认和最终关闭继续留给人工签字。
9. 所有示例均标注为 `模拟数据`；不新增后端代码，不接入任何真实系统或真实数据。

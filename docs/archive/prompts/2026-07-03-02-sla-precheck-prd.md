# SLA 预诊助手一期 PRD 生成提示词归档

## 使用时间

2026/07/03

## 使用目的

用于生成一期 `SLA 预诊助手` PRD，并交付 Markdown 版和 HTML 评审展示页，同时更新项目进度、路线说明和变更记录。

## 适用范围

适用于 `Transwarp Service Insight` 项目内一期 `SLA 预诊助手` 的需求整理、产品评审、试点范围讨论和后续任务拆解。

## 提示词正文

请基于以下计划生成一期 `SLA 预诊助手` PRD：

- PRD 范围只写一期 `SLA 预诊助手`，二阶段 `智能工单助手` 仅作为后续方向简述。
- 核心指标优先衡量 `提交质量提升`，SLA 数量下降只作为间接收益，不写成强压指标。
- 一期形态定义为 `可试点系统`，不是纯前端 Demo，也不是生产全量上线。
- 公司已有 AIOps 系统，支持历史 SLA 查询和 SLA 提交。
- 一期建设要求预留 AIOps 对接的产品接口要求，不写具体 API 协议、接口地址或后端实现。
- 一期知识来源优先接入售后知识库、Wiki、内部培训素材、使用手册和处理手册。
- 提交流程坚持 `建议但不阻断`，预诊助手给出建议、依据、置信度和待补充信息，但员工仍可继续提交 SLA。
- HTML 版本定位为 `评审展示页`，做成可直接打开的静态单页文档，适合会议投屏和浏览。

需要交付：

- 新增 `docs/prd-sla-precheck-assistant.md`，作为 PRD 内容主文档。
- 新增 `docs/prd-sla-precheck-assistant.html`，作为可直接打开的评审展示页。
- HTML 使用静态单页，不接接口、不加载外部资源、不依赖构建工具。
- 可选更新 `docs/progress.md` 和 `docs/roadmap.md`，记录 PRD 编写进展和 AIOps 既有系统的预留接口说明。
- 更新 `CHANGELOG.md`，记录新增 PRD Markdown/HTML、AIOps 对接接口预留和范围口径调整。

必须遵守：

- 不生成后端代码。
- 不调用真实接口。
- 不接入真实 RAG、真实大模型、真实 AIOps、真实 Wiki 或真实历史 SLA。
- 不接入数据库。
- 不自动提交 SLA，不自动闭单。
- 不把智能预诊结果写成最终根因或最终处理结论。
- 不使用未脱敏的真实客户、真实日志、真实 SLA 或真实内部链接。
- 所有涉及示例的内容必须明确标注为 `模拟数据`。
- 所有模型输出必须包含依据来源、置信度、人工介入建议和待补充信息。

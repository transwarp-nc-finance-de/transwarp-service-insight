# 验收测试计划

## 目的

本文档定义从当前 Demo 到后续目标架构的验收测试计划。

## 适用范围

适用于阶段验收、发布前评审和后续试点评估。当前阶段只验收文档、Demo 和 mock 数据。

## 正文

当前阶段验收：Demo 页面可用、文档目录清晰、架构设计完整、所有示例标注 `模拟数据`、不声称真实系统接入。

后续阶段验收：API 草案评审、RAG 原型评估、LLM 生成与安全评审、ITSM / AIOps 集成评审、降级容灾测试、审计与权限测试。

所有阶段均需保留人工审核，不自动提交 SLA，不自动闭单，不输出最终根因。

### 来源配置、附件与开放知识专项验收

1. 默认四类来源和通用知识降级均开启，默认 Foundation 场景仅展示实际命中的内部模拟资料。
2. 分别关闭 Wiki、历史 SLA、产品手册、开源资料，验证禁用来源不出现在 loading、候选卡、引用、命中统计和回答正文。
3. RAG 关闭且通用知识降级开启时，只返回低置信度通用建议；所有来源和降级均关闭时，只提示信息不足和人工继续提交 SLA。
4. 保存配置后刷新恢复；预诊开始后再改配置，已有对话继续展示原策略版本和来源快照。
5. 将表单调整为 Kafka/KRaft 元数据仲裁超时场景，验证内部无命中后仅命中已开启的开源模拟资料。
6. 验证模拟截图、模拟日志的添加、预览、删除、随消息发送，以及只读命令复制。
7. 回归 loading、0–4 类启用来源、四类快捷补充、自由输入、继续提交、确定和取消。
8. Chromium 已完成技术复验；第一阶段 Edge 冒烟和业务关闭已由授权责任人确认。后续真实来源准入与管理员责任仍须单独人工评审。

验收证据必须继续标注 Demo/模拟数据边界，不得据此声称已接入真实系统、资料、模型或互联网。

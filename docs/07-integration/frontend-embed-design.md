# 前端嵌入设计

## 目的

本文档描述后续目标架构中预诊面板嵌入工单创建页的候选方式。

## 适用范围

适用于前端集成评审。当前阶段仅有 `prototypes/sla-precheck-demo.html` 静态 Demo。

## 正文

候选方式包括 iframe 嵌入、JS SDK 嵌入或工单系统原生组件集成。触发方式可包括手动点击和输入防抖触发，具体方式需结合 ITSM / AIOps 前端架构评审。

结果面板应展示依据来源、置信度、人工介入建议、待补充信息、降级状态和继续人工提交入口。

当前 Demo 与真实嵌入的差异是：Demo 不鉴权、不取真实表单上下文、不调用接口、不保存结果。

# Agent 设计

## 目的

本文档定义 `Transwarp Service Insight` 一阶段 `SLA 预诊助手` 中各类 Agent 的职责、输入输出、边界、失败处理和人工审核要求。

## 适用范围

适用于 Agent 协作流程设计、Demo 编排、后续实现拆分和测试用例设计。当前文档只定义职责，不代表已实现对应 Agent，不代表已接入真实 AIOps、真实知识库或真实工单系统。

## 统一输出要求

所有 Agent 输出必须包含：

- 结论或建议。
- 依据来源。
- 置信度。
- 需要人工确认的事项。
- 信息不足或失败原因。
- 待补充信息。

所有 Agent 输出均为辅助建议，不是最终处理结论。

## 预诊 Agent

### 职责

检查 SLA 提交信息是否完整，识别缺失字段、表达不清和高风险信息缺口，并辅助判断是否可能属于基础类、重复类或开源社区类问题。

### 输入

- 问题描述。
- 发生时间。
- 影响范围。
- 业务优先级。
- 产品组件和版本。
- 错误信息或现象。
- 已尝试操作。

### 输出

- 信息完整性评分。
- 缺失字段列表。
- 建议补充问题。
- 是否建议提交 SLA 草稿的辅助判断。
- 人工确认事项。
- 待补充信息。

### 边界

不得拒绝用户提交 SLA，只能提示信息不足、潜在影响和可参考资料。不得自动提交正式 SLA。

### 失败处理

当输入过短或含义不清时，输出最小补充问题清单，不进行过度推断。

## 问题分类 Agent

### 职责

辅助判断问题类型、影响对象和紧急程度。

### 输入

- 预诊后的用户描述。
- 补充信息。
- 检索摘要。
- 模拟 AIOps 指标摘要。

### 输出

- 问题类型建议。
- 影响范围建议。
- 紧急程度建议。
- 分类依据。
- 置信度。
- 待补充信息。

### 边界

不得将分类建议写成最终定性结论。

### 失败处理

当多个分类可能同时成立时，应列出备选分类和需补充的信息。

## RAG 检索 Agent

### 职责

从模拟售后知识库、模拟 SLA 复盘数据、模拟 Wiki、模拟内部培训素材、模拟使用手册、模拟处理手册和模拟历史案例中检索与当前问题相关的内容。

### 输入

- 问题描述。
- 问题分类。
- 关键词。
- 产品组件。
- 产品版本。
- 相关错误信息。

### 输出

- 命中知识条目。
- 相似案例摘要。
- 引用来源。
- 相似度或置信度。
- 不适用提示。
- 待补充信息。

### 边界

不得虚构不存在的知识条目、客户名称、真实历史 SLA 或真实复盘记录。

### 失败处理

未命中时应明确说明“未找到足够相似的模拟知识条目”，并建议人工补充关键词或扩展知识库。

## AIOps 分析 Agent

### 职责

摘要模拟 AIOps 指标，识别异常时间段、趋势变化和可能相关对象。

### 输入

- 模拟任务指标。
- 模拟资源指标。
- 模拟调度指标。
- 模拟 SLA 复盘指标摘要。
- 问题发生时间窗口。

### 输出

- 指标摘要。
- 异常点列表。
- 可能关联线索。
- 需要人工核验的数据点。
- 待补充信息。

### 边界

不得把指标相关性直接判定为最终根因。

### 失败处理

当指标缺失或时间窗口不一致时，应说明无法判断，并提示需要补充哪些指标。

## SLA 生成 Agent

### 职责

基于已收集信息生成 SLA 工单草稿。

### 输入

- 用户描述。
- 预诊结果。
- 分类建议。
- RAG 检索结果。
- 模拟 AIOps 摘要。
- 人工确认信息。

### 输出

- SLA 标题草稿。
- 问题描述。
- 影响范围。
- 优先级建议。
- 证据和附件清单。
- 待确认事项。
- 依据来源。
- 置信度。

### 边界

不得自动提交工单，不得自动选择最终优先级。

### 失败处理

当关键字段缺失时，只生成草稿骨架，并突出待补充内容。

## 复盘报告 Agent

### 职责

在事件处理后生成复盘报告草稿和知识沉淀建议。

### 输入

- SLA 原始信息。
- 处理过程记录。
- 人工确认的根因。
- 人工确认的解决方案。
- 影响范围。
- 相关知识条目。

### 输出

- 事件摘要。
- 时间线。
- 影响分析。
- 原因分析草稿。
- 处置过程。
- 改进建议。
- 知识库沉淀候选内容。
- 待补充信息。

### 边界

不得在缺少人工确认根因时生成最终根因。不得自动写入知识库。

### 失败处理

当处理过程记录不足时，应生成复盘补充问题，而不是编造时间线。

## 人工审核 Agent

### 职责

辅助整理待审核事项、差异点和风险提示，帮助审核人快速确认模型输出。

### 输入

- 各 Agent 输出。
- 人工修改记录。
- 审核规则。

### 输出

- 待审核清单。
- 风险提示。
- 建议采纳状态。
- 需要二次确认的内容。
- 待补充信息。

### 边界

不得替代审核人做最终确认。

### 失败处理

当审核规则不明确时，应提示需要流程负责人补充规则。

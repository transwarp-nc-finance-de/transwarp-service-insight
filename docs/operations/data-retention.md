# 数据保留与删除

Status: DRAFT  
Owner: 数据负责人 / 安全负责人  
Last reviewed: 2026-07-12  
Source of truth for: 未来持久化前的数据治理门禁

## 当前实现

预诊会话仅存在浏览器页面内；反馈、知识领域样例和审计事件只存在当前后端进程内。刷新或进程重启会清空相应状态，没有备份、恢复、导出或正式删除接口。

## 持久化前必须确认

- 每类数据的业务目的、责任人、法律依据与来源授权；
- 输入快照、Evidence、反馈、审计、知识原文和索引的独立保留期限；
- 用户/项目删除、知识废弃、索引清理、备份过期和法务保全流程；
- 脱敏、加密、密钥管理、访问审计、地域及外部模型传输边界；
- 删除完成证据、失败重试、前向修复与恢复演练。

默认原则是最少收集、最短必要保留、正文与结构化元数据分离。未完成上述确认前不得接入真实数据库或真实数据。

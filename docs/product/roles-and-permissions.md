## 角色

| 角色 | 允许能力 | 禁止能力 |
| --- | --- | --- |
| SLA 提交人 | 填写问题、触发预诊、补充脱敏信息、查看有权依据、反馈、人工确认提交 | 扩大知识来源范围、自动执行建议、跳过人工确认 |
| SLA 处理人 | 查看授权范围内的预诊、依据与人工反馈 | 将预诊直接认定为最终根因或正式结论 |
| 知识维护人员 | 导入候选知识、编辑元数据、提交审核 | 自行批准本人提交的敏感知识、绕过权限范围 |
| 知识审核人员 | 审核、批准、拒绝、发布或废弃授权范围内版本 | 发布未授权、未脱敏或来源不明内容 |
| 运营人员 | 查看聚合使用与效果指标 | 查看无业务必要的输入正文或越权资料 |
| 系统管理员 | 配置身份、角色、来源白名单和策略 | 替代业务人员确认 SLA 内容或自动提交 |
| 审计/安全人员 | 查看安全事件、策略版本和结构化审计 | 默认读取问题、附件或反馈全文 |

## 一期已确认本地职责分离

- `KNOWLEDGE_EDITOR` 与 `KNOWLEDGE_REVIEWER` 必须对应不同的本地模拟身份，而不只是同一身份切换角色。
- 提交知识版本的身份不得批准或发布自己的版本；一名不同身份的审核人即可完成审核，不要求双人会签。
- 职责分离由后端授权规则执行，前端按钮可见性不能作为唯一控制。

以上为 `CONFIRMED` 的一期目标权限规则。不同的编辑与审核模拟身份已预置；知识草稿修订、送审、退回、批准和职责分离业务路径已实现，发布与废弃业务路径尚未实现。

## 当前实现边界

当前已实现 `local-identity-v1` 的四个模拟身份、服务端 Cookie Session、CSRF、角色与 TDH/STREAMING 产品线矩阵；只适用于本地模拟数据，不构成真实认证或生产授权。v1 预诊仍使用 `mock-policy-v1` 和进程内状态。

## 一期已确认本地身份方式

一期提供预置 `模拟数据` 用户的本地 Mock 登录/切换，使用服务端 Session/Cookie。后端必须依据当前身份执行 RBAC 并将主体写入审计；不提供用户注册、找回密码、真实组织同步、JWT 签发或正式 SSO。未来身份来源通过 `IdentityContextPort` 替换。

一期固定使用 `PRECHECK_USER`、`KNOWLEDGE_EDITOR`、`KNOWLEDGE_REVIEWER`、`ADMIN` 四个角色。`ADMIN` 可查看基本评估与结构化审计、执行受控数据重置，但不能审核自己提交的版本、替代知识审核人或替预诊用户确认继续提交。

`ADMIN` 可查看不含正文的本地运行聚合摘要，包括预诊、降级、多轮、引用、反馈、继续提交和延迟指标；不得通过聚合页面反查无业务必要的用户输入。

`ADMIN` 可只读查看完整度策略及版本，但不能通过 UI 修改或发布规则。一期策略变更由受评审的版本化配置与初始化流程完成。

一期数据范围采用“角色 + 预置产品线范围”。知识 Chunk 继承所属文档的产品线范围；检索必须先按身份和产品线过滤，再执行关键词或向量召回。无权限资料不得进入候选集、引用、生成上下文或非必要审计摘要。一期不提供组织树、任意文档 ACL 或用户组管理。

`PRECHECK_USER` 只能恢复本人未终止且仍可补充的 Session，终态 Session 只读；恢复时重新校验当前产品线权限。`ADMIN` 可查看授权范围内的结构化审计，但不得接管或恢复其他用户的 Session。

## 未来授权计算

有效知识范围 = 管理员允许来源 ∩ 用户角色/组织/项目范围 ∩ 文档及 Chunk 权限 ∩ 已发布版本 ∩ 产品版本适用范围 ∩ 查询过滤条件。

真实 SSO、组织结构、客户/项目范围、管理员职责分离和紧急授权流程必须由安全及业务负责人确认后实施。

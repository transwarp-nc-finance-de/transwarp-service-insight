# 产品与评估指标

Status: DRAFT  
Owner: 产品运营负责人  
Last reviewed: 2026-07-20
Source of truth for: 当前工程评估、聚合指标口径与归因规则

## 业务指标

- SLA 字段完整度、预诊后信息补充率、反馈率与建议采纳率；
- 引用点击率、自助解决率、继续提交率与有效 SLA 占比；
- 首次分派/响应耗时、处理人追问次数和知识缺口转化率。

## 检索与生成指标

- Recall@K、MRR、NDCG、引用正确率、证据覆盖率与回答忠实度；
- 拒答准确率、权限泄漏率、过期知识命中率和预诊误导率；
- P50/P95/P99 延迟、超时/降级率、单次 Token 与成本。

## 归因规则

不得仅以“SLA 数量下降”判断成功。下降也可能来自用户放弃、误导、系统不可用或入口使用率下降，必须同时观察字段完整度、有效 SLA、用户反馈、处理耗时和错误引导率。

当前已有本地 PostgreSQL 持久化的预诊 Session/Run、Retrieval/Evidence、Feedback、SubmissionContinuation、结构化 AuditEvent、`EvaluationRun` 和案例结果；Metrics API 从这些实际持久化事件聚合，管理员评估页提供最小入口。这些本地 `模拟数据` 只能支持工程行为验证，不代表生产业务或模型效果；任何生产基线、目标值与统计周期仍须由产品、SLA 处理人及安全负责人确认。

## 一期已确认反馈口径

采纳行为与有用性评价分别统计：采纳行为为 `ADOPTED / PARTIALLY_ADOPTED / IGNORED`，有用性评价为可选的 `HELPFUL / NOT_HELPFUL`。未提供有用性评价不得被计为 `NOT_HELPFUL`，反馈失败或缺失也不得推断用户继续提交的原因。

## 一期已实现评估入口

`ADMIN` 可通过最小评估页手动运行固定评估集，查看最近一次及历史结果摘要，并通过 API 分页读取安全失败摘要；评估运行、30 条案例结果及实际 v2 Session/Run/Retrieval/Evidence 轨迹均持久化。EvaluationRun 与 Metrics operation 已标记 `IMPLEMENTED`；一期不提供实时运营大屏、A/B、告警或复杂趋势分析。

固定评估集不少于 30 条 `模拟数据` 策划样例，覆盖精确词、语义改写、证据不足、权限隔离、Embedding 降级、多轮补充与引用核验。每条记录期望 Evidence、允许范围和预期降级或待补充信息。所有汇总结果必须声明“小样本工程评估，不代表生产效果”。

Embedding 评估必须覆盖中文、英文和中英混合技术文本，以及中英文资料互检索；错误码、版本号和配置键的精确命中单独由 FTS/混合检索样例验证。

默认 `local-embedding` 的资源门槛为：模型制品不超过 2 GB、服务内存不超过 4 GB、CPU 可运行且无需 GPU；并发 1 时单查询 P95 不超过 1 秒。批量建索引单独记录吞吐，不使用单查询延迟门槛。

默认评估对象固定为 `intfloat/multilingual-e5-base@d13f1b27baf31030b7fd040960d60d909913633f`，768 维、512 Token，检索输入使用 `query:`/`passage:` 前缀。模型变更必须产生新的模型和索引版本，不得与旧评估结果混用。

目标最小页面展示本地运行聚合摘要：预诊次数、成功/降级率、平均 Run 数、信息补充率、引用命中率、采纳率、继续提交率，以及检索和 Embedding 延迟。聚合不得读取或展示用户输入正文，不用于自动判责，也不设置业务收益硬门槛。

一期工程验收门槛：权限泄漏率 0%、引用错误率 0%、降级场景通过率 100%、Recall@5 不低于 80%。前 3 项为硬门禁；Recall@5 低于门槛时不得把该版本标记为一期评估通过。这些门槛不外推为生产 SLA、业务收益或模型效果承诺。

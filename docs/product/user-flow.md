# 用户流程

## 目的

本文档描述 `SLA 智能预诊助手` 从用户填写问题到查看预诊建议、人工确认并选择后续动作的流程。

## 适用范围

适用于当前技术 MVP 评审、一期 PRD 评审和后续可试点系统设计。当前能力调用本地 Mock API，不自动提交 SLA、不自动闭单且不接真实系统。

## 正文

### 当前技术 MVP 流程

```text
填写模拟 SLA 表单
-> 点击智能预诊
-> 展示预诊中 loading
-> 展示模拟预诊结果
-> 查看依据来源、置信度、人工介入建议和待补充信息
-> 使用快捷入口或自由输入进行页面内模拟多轮追问
-> 查看建议下一步，且始终可继续人工提交
-> 人工判断是否继续提交 SLA
```

### 后续目标流程

```text
填写问题
-> 触发预诊
-> 展示建议
-> 采纳并自行处理
-> 部分采纳并继续提交
-> 忽略并直接提交
```

### 一期本地完整纵向闭环（CONFIRMED 目标）

```text
KNOWLEDGE_EDITOR 本地登录
-> 上传模拟、公开或脱敏的 Markdown/TXT/文本型 PDF
-> 查看解析、警告与结构化切片预览
-> 必要时创建草稿修订
-> 提交审核
-> 不同身份的 KNOWLEDGE_REVIEWER 审核
-> FTS 与向量索引都成功后原子发布
-> PRECHECK_USER 选择本人产品线范围内的模拟问题
-> 创建或恢复 PrecheckSession
-> 执行完整度检查、权限过滤、FTS/向量召回与 RRF 融合
-> 确定性模板输出模拟建议、Evidence、置信度理由、人工介入建议与待补充信息
-> 用户最多补充两轮，或随时跳过补充
-> 用户独立记录反馈
-> 用户确认自助结束，或记录模拟继续提交
-> Feedback、SubmissionContinuation 与 AuditEvent 分别持久化
-> ADMIN 手动运行固定评估集并查看聚合摘要与失败案例
```

用户在任何置信度、信息完整度、Run 上限或能力降级状态下都可以继续人工流程。反馈失败不得影响模拟继续提交；闭环不创建 SLA、工单草稿或提交回执。

一期闭环结束后，`ADMIN` 可在最小评估页手动运行固定评估集，查看持久化的历史结果摘要和失败案例；该页面不是完整运营后台。

### 状态说明

| 状态 | 说明 | 人工要求 |
| --- | --- | --- |
| 草稿填写 | 用户填写标题、描述、组件、版本、级别和影响范围 | 用户负责输入准确性 |
| 触发预诊 | 用户点击按钮 | 当前技术 MVP 仅手动触发 |
| 结果展示 | 展示资料、建议补充信息和风险提示 | 结果只作辅助建议 |
| 采纳 | 用户认为资料可帮助自助处理 | 由用户人工确认，不自动闭单 |
| 部分采纳 | 用户继续提交并附带预诊摘要 | 由用户人工确认，不自动提交 |
| 忽略 | 用户不使用建议并继续提交 | 系统不得阻塞提交 |

所有示例流程均为 `模拟数据` 语境，不代表真实客户、真实 SLA 或真实系统行为。

# 用户流程

## 目的

本文档描述 `SLA 智能预诊助手` 从用户填写问题到查看预诊建议、人工确认并选择后续动作的流程。

## 适用范围

适用于第一阶段纯前端 Demo 评审、一期 PRD 评审和后续可试点系统设计。当前 Demo 仅模拟流程，不自动提交 SLA，不自动闭单，不接真实系统。

## 正文

### 当前 Demo 流程

```text
填写模拟 SLA 表单
-> 点击智能预诊
-> 展示预诊中 loading
-> 展示模拟预诊结果
-> 查看依据来源、置信度、人工介入建议和待补充信息
-> 人工判断是否继续提交 SLA
```

### 后续目标流程

```text
填写问题
-> 触发预诊
-> 展示建议
-> 采纳并自行处理
-> 部分采纳并继续提交
-> 忽略并直接提交
```

### 状态说明

| 状态 | 说明 | 人工要求 |
| --- | --- | --- |
| 草稿填写 | 用户填写标题、描述、组件、版本、级别和影响范围 | 用户负责输入准确性 |
| 触发预诊 | 用户点击按钮或后续阶段通过防抖触发 | 当前 Demo 仅手动触发 |
| 结果展示 | 展示资料、建议补充信息和风险提示 | 结果只作辅助建议 |
| 采纳 | 用户认为资料可帮助自助处理 | 由用户人工确认，不自动闭单 |
| 部分采纳 | 用户继续提交并附带预诊摘要 | 由用户人工确认，不自动提交 |
| 忽略 | 用户不使用建议并继续提交 | 系统不得阻塞提交 |

所有示例流程均为 `模拟数据` 语境，不代表真实客户、真实 SLA 或真实系统行为。

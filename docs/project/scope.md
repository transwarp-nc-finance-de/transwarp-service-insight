# 当前范围：技术 MVP

## 目的

提供当前阶段唯一的短版范围入口，统一说明已完成内容、M1/M2 目标和边界。

## 适用范围

适用于技术 MVP 的开发、评审、演示和验收。

## 正文

第一阶段纯前端 `SLA 智能预诊助手` Demo 已于 2026/07/10 关闭，历史原型归档在 `docs/archive/demo-v1/`。技术 MVP M1 已于 2026/07/12 完成交付。M2 在相同 Mock 闭环上增加本地工程化交付：多阶段镜像、仅暴露前端的 Compose、进程健康接口、OpenAPI 门禁与容器冒烟。下一阶段真实能力仅为候选工作，尚未获得批准。

M1 已交付 SLA 表单、`POST /api/v1/precheck`、确定性模拟预诊、结构化错误、结果面板、人工继续提交模拟、前后端测试和启动文档。当前在此闭环上新增 `POST /api/v1/precheck/follow-up` 与页面内多轮追问：会话刷新即清空，后端不校验或持久化会话，仅按日志/报错、权限、影响范围、近期变更关键词返回确定性 Mock，未命中时返回低置信度信息不足提示。所有参考资料和结果均为 `模拟数据`；不接数据库、真实 RAG、真实模型、AIOps、ITSM、Wiki、历史 SLA 或生产数据。

多轮追问不会自动回填或修改 SLA 表单，失败不会清空既有历史或阻断人工继续提交；每轮回复均包含模拟依据、置信度、人工介入建议和待补充信息。SLA 是否提交及提交内容始终由人工确认。

当前架构骨架增加 `POST /api/v1/precheck/feedback`、模拟身份策略快照与内存审计 Adapter。反馈仅保留在当前后端进程，重启即清空；审计只记录事件类型、模拟主体、策略版本、置信度/状态和人工动作等结构化元数据，不记录问题描述、追问或反馈正文。该骨架不代表真实 SSO、权限系统、审计平台或数据库已接入。

当前非目标包括真实鉴权、持久化、文件上传、真实提交、生产部署、容器热更新开发、自动闭单、最终根因判断和知识库沉淀。建议仅供人工参考，反馈记录失败、预诊失败或信息不足均不得阻断人工继续提交。

旧 PRD、架构、RAG、安全、API 和门禁文档继续作为历史基线或后续目标架构参考，不代表其中能力已实现。归档的下一阶段门禁材料仅供真实试点阶段参考，不代表已进入试点设计或已获得实施授权。

数据仅限模拟、公开或已脱敏内容。涉及真实数据、外部服务、破坏性操作、公共接口兼容或重大业务歧义时必须人工确认。

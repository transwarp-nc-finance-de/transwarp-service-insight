# Demo 计划

## 目的

本文档定义 `SLA 智能预诊助手` 第一阶段纯前端 Demo 的演示目标、主场景、对话式流程、模拟数据边界和验收方式。

## 适用范围

适用于内部演示、方案评审、文档交接和原型验收。本文所有字段、问题描述、参考资料、链接、对话回复和结论均为 `模拟数据`。

## 正文

### 演示目标

展示用户在提交 SLA 前，通过 `智能预诊` 入口进入对话式预诊抽屉，由前端基于当前表单字段和静态 mock 资料生成初步辅助建议，并支持通过快捷反馈或自由输入补充日志、权限、影响范围和近期变更信息。

Demo 目标不是自动解决 SLA，而是验证提交前辅助入口是否清晰、对话式补充信息是否自然、低置信度追问是否能改善工单质量、人工审核边界是否充分。

### 定稿范围

当前定稿范围仅包含 `prototypes/sla-precheck-demo.html`、前端静态 mock 数据、演示文档和验收说明。不包含后端代码、真实接口、真实 RAG、真实大模型、真实 AIOps、真实 Wiki、真实历史 SLA 或数据库接入。

### Demo 主场景

模拟数据：

- 产品组件：Foundation。
- 产品版本：TDS-4.0.1。
- 问题级别：Sev3。
- 服务类型：现场问题解决。
- 问题概述：Foundation 审计日志归档功能不可用。
- 问题描述：用户在 Foundation 页面执行审计日志归档时，归档功能不可用或按钮置灰，无法完成归档操作。

### 页面流程

1. 打开 `prototypes/sla-precheck-demo.html`。
2. 展示模拟 SLA 报备页面。
3. 说明页面字段均为 `模拟数据`。
4. 点击底部 `智能预诊`。
5. 展示 loading，并说明使用前端 `setTimeout` 和模拟数据，不调用真实接口。
6. 等待 1.5 到 3 秒。
7. 展示 `对话式智能预诊` 抽屉。
8. 讲解自动生成的表单摘要、初步助手回复和 4 条模拟参考资料。
9. 依次演示快捷反馈：补充日志现象、补充权限信息、补充影响范围、补充近期变更。
10. 输入无关文本，演示低置信度追问。
11. 点击 `继续提交 SLA`，说明该动作只关闭抽屉并提示人工确认。
12. 点击原页面 `确定`，展示模拟提交提示。

### 对话内容要求

初步助手回复必须包含：

- 诊断结论：仅为辅助建议，不代表最终根因或处理结论。
- 依据来源：页面表单字段、前端静态 mock 知识数组、模拟参考资料。
- 置信度：中。
- 人工介入建议：需要提交人和处理人确认。
- 待补充信息：日志、权限、影响范围、近期变更等。

多轮回复通过前端静态关键词分支生成。每轮回复都必须继续标注 `模拟数据`、`仅供参考`、`需人工确认`，不得宣称命中真实知识库或完成真实诊断。

### 参考资料要求

推荐资料固定为 4 条模拟资料卡，覆盖 Wiki、历史问题、历史 SLA 和产品文档。每条摘要以 `模拟数据` 开头，链接使用 `example.com` 示例域，不连接真实内部系统。

### 人工审核提示

演示时必须说明：

- SLA 是否提交必须由人工确认。
- 工单内容是否准确必须由人工确认。
- 问题分类、影响范围和 Sev 级别必须由人工确认。
- 处理建议是否适用于当前环境必须由人工确认。
- 智能预诊内容不能作为最终根因、最终处理方案或正式复盘结论。

### 边界说明

- 当前 Demo 不接入真实后端。
- 当前 Demo 不接入真实 RAG。
- 当前 Demo 不接入真实大模型。
- 当前 Demo 不接入真实 AIOps。
- 当前 Demo 不接入真实 Wiki 或 SLA 系统。
- 当前 Demo 不自动提交 SLA。
- 当前 Demo 不自动闭单。
- 当前 Demo 不输出最终根因或最终处理结论。

### 验收方式

通过浏览器打开 HTML 文件并完成一次完整对话流程。验收重点是交互闭环、文案一致性、模拟数据标注、关键词分支可用性、低置信度追问和人工审核边界，不评估真实检索准确率。

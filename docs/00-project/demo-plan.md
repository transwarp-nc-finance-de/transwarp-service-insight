# Demo 计划

## 目的

本文档定义 `SLA 智能预诊助手` 第一阶段纯前端 Demo 的演示目标、主场景、页面流程、结果内容、边界提示和验收方式。

## 适用范围

适用于内部演示、方案评审、文档交接和原型验收。本文所有字段、问题描述、参考资料、链接和结论均为 `模拟数据`。

## 正文

### 演示目标

展示在模拟 AIOps SLA 报备页面中，用户提交 SLA 前可以点击 `智能预诊`，查看系统基于当前表单信息返回的模拟参考资料、前置诊断结论和建议补充信息。

Demo 目标不是自动解决 SLA，而是验证提交前辅助入口是否清晰、结果边界是否明确、人工审核提示是否充分。

### Demo 主场景

模拟数据：

- 产品组件：Foundation。
- 产品版本：TDS-4.0.1。
- 问题级别：Sev3。
- 服务类型：现场问题解决。
- 问题概述：Foundation 审计日志归档功能不可用。
- 问题描述：用户在 Foundation 页面执行审计日志归档时，归档功能不可用或按钮置灰，无法完成归档操作。

### 页面流程

1. 打开 `prototypes/sla-precheck-demo.html`。
2. 展示模拟 SLA 报备页面。
3. 说明页面字段均为 `模拟数据`。
4. 点击底部 `智能预诊`。
5. 展示 loading：`正在检索 Wiki、历史问题、历史 SLA、操作手册和产品文档，请稍候...`。
6. 等待 1.5 到 3 秒。
7. 展示 `智能预诊结果`。
8. 讲解检索关键词、检索范围、前置诊断结论、建议补充信息和推荐参考资料。
9. 点击 `继续提交 SLA`，说明该动作只关闭面板并提示人工确认。
10. 点击原页面 `确定`，展示模拟提交提示。

### 结果内容

智能预诊结果必须包含：

- 检索关键词：Foundation、TDS-4.0.1、审计日志归档、功能不可用。
- 检索范围：Wiki、历史问题、历史 SLA、操作手册、产品文档。
- 前置诊断结论：当前仅基于模拟知识库返回相关资料，暂未发现可以直接解决该问题的明确方案。
- 建议补充信息：权限、后台服务日志、近期升级或配置变更、归档路径、存储空间和权限异常。
- 推荐参考资料：4 条模拟资料。
- 操作提示：如资料不能解决当前问题，请继续提交 SLA。

### 人工审核提示

演示时必须说明：

- 依据来源：页面表单字段与前端模拟知识库数据。
- 置信度：中。
- 是否需要人工介入：需要。
- 待补充信息：真实环境日志、复现步骤、影响范围、权限与配置核查结果。

### 边界说明

- 当前 Demo 不接入真实后端。
- 当前 Demo 不接入真实 RAG。
- 当前 Demo 不接入真实大模型。
- 当前 Demo 不接入真实 AIOps。
- 当前 Demo 不接入真实 Wiki 或 SLA 系统。
- 当前 Demo 不自动提交 SLA。
- 当前 Demo 不自动闭单。
- 当前 Demo 不输出最终根因或最终处理结论。

### 后续可扩展场景

以下场景仅作为后续演进，不作为第一阶段主 Demo：

- 模拟数据：凌晨跑批任务延迟，部分任务长时间等待，影响日报生成。
- 模拟数据：数据同步失败导致下游报表延迟。
- 模拟数据：队列资源使用率异常导致任务等待。

### 验收方式

通过浏览器打开 HTML 文件并完成一次点击流程。验收重点是交互闭环、文案一致性、模拟数据标注和人工审核边界，不评估真实检索准确率。

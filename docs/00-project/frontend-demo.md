# 前端 Demo 交互说明

## 目的

本文档说明 `SLA 智能预诊助手` 第一阶段纯前端 Demo 的页面入口、对话式智能预诊流程、loading 效果、多轮反馈、mock 数据边界和验收标准。

## 适用范围

适用于 `prototypes/sla-precheck-demo.html` 的原型评审、演示讲解和后续静态页面维护。本文只描述纯前端静态 Demo，不包含真实后端、真实 RAG、真实大模型、真实 AIOps、真实 Wiki、真实历史 SLA 或数据库接入。

## 正文

### 页面改造目标

在模拟 AIOps SLA 报备页面中保留底部 `智能预诊` 按钮。用户提交 SLA 前点击该按钮后，抽屉不再展示一次性结果面板，而是展示类似 GPT 的对话式智能预诊。

对话内容全部为 `模拟数据`。助手回复仅供参考，不代表最终根因或最终处理结论，SLA 是否提交、内容是否准确、建议是否适用于当前环境必须由人工审核。

### 点击流程

1. 用户查看 Foundation 审计日志归档功能不可用的模拟 SLA 表单。
2. 点击 `智能预诊`。
3. 按钮变为 `预诊中...` 并禁用重复点击。
4. 抽屉打开并展示 loading。
5. 前端使用本地 `setTimeout` 模拟 1.5 到 3 秒耗时。
6. loading 结束后展示 `对话式智能预诊`。
7. 用户可点击快捷反馈或输入补充信息触发下一轮模拟回复。
8. 点击 `继续提交 SLA` 只关闭抽屉并提示人工确认，不自动提交。
9. 原页面 `确定` 仍只显示模拟提交提示。

### 对话式智能预诊

初始对话包含两条消息：

- 用户消息：由当前 SLA 表单字段自动生成摘要，标注为 `模拟数据`。
- 助手消息：说明已基于表单字段和模拟资料完成初步预诊。

初步助手回复必须包含：

- 诊断结论：仅为提交前辅助建议，不代表最终根因或处理结论。
- 依据来源：页面组件、版本、问题概述和问题描述、前端本地关键词规则，以及实际命中的 `example.com` 模拟参考资料数量。
- 置信度：有资料命中时为中；无命中时为低。
- 人工介入建议：提交人和处理人必须确认。
- 待补充信息：日志时间点、组件日志路径、审计管理权限、影响范围、近期升级或配置变更。

### 推荐参考资料

对话中的候选资料收敛为 Wiki、历史 SLA、产品手册三类，每类最多一条。前端只根据组件、版本、问题概述和问题描述执行本地确定性关键词过滤，按实际命中展示 0–3 条，不为凑数量展示无关资料。所有摘要必须标注 `模拟数据`，所有链接必须使用 `example.com` 示例域。

有部分命中时只渲染实际命中的资料卡；无命中时不渲染资料标题、卡片容器或占位卡片，只显示“暂未检索到相关模拟资料，可补充信息或继续提交 SLA”。无命中不代表问题没有解决方案，也不阻止人工继续提交 SLA。

### 多轮反馈

抽屉底部提供自由输入框、发送按钮和 4 个快捷反馈按钮：

- `补充日志现象`
- `补充权限信息`
- `补充影响范围`
- `补充近期变更`

用户输入或点击快捷按钮后，前端追加用户消息，并通过静态 JS 关键词分支返回下一轮模拟助手回复。

### 关键词分支规则

- 命中 `日志`、`报错`、`error`、`exception` 或 `log`：提示继续补充日志时间点、组件日志路径、完整报错上下文和复现步骤，置信度为中。
- 命中 `权限`、`角色`、`审计管理`、`授权`、`permission` 或 `role`：提示核查审计管理权限、角色配置和功能开关，置信度为中。
- 命中 `影响范围`、`全员`、`单用户`、`集群`、`范围`、`severity` 或 `sev`：提示确认全员、单用户或单集群影响，并建议人工判断 SLA 级别，置信度为中。
- 命中 `变更`、`升级`、`配置`、`调整`、`change`、`upgrade` 或 `config`：提示核查近期变更记录和配置差异，置信度为中。
- 未命中：提示信息仍不足，要求按日志、权限、影响范围、近期变更继续补充，置信度为低。

所有分支回复都必须包含 `模拟数据`、依据来源、置信度、人工介入建议和待补充信息，不得写成最终处理结论。

### 交互边界

- `智能预诊` 不自动提交 SLA。
- `智能预诊` 不自动闭单。
- `智能预诊` 不判断最终根因。
- `智能预诊` 不生成最终处理方案。
- `智能预诊` 不访问真实接口。
- `继续提交 SLA` 只关闭抽屉并提示人工确认。
- 原页面 `确定` 按钮保持可用，但当前 Demo 只做模拟提交提示。

### 验收标准

1. `prototypes/sla-precheck-demo.html` 可直接在浏览器打开。
2. 点击 `智能预诊` 后按钮变为 `预诊中...`。
3. loading 持续 1.5 到 3 秒。
4. 抽屉标题为 `对话式智能预诊`，并标注 `模拟数据`。
5. 初始用户消息自动来自当前 SLA 表单摘要。
6. 初始助手消息包含诊断结论、依据来源、置信度、人工介入建议和待补充信息。
7. 默认 Foundation 场景命中 Wiki、历史 SLA、产品手册三类模拟资料；部分命中时只展示 1–2 条，无命中时只展示无相关模拟资料提示，链接均为 `example.com`。
8. 快捷反馈和自由输入均能追加用户消息并触发模拟回复。
9. 日志、权限、影响范围、近期变更关键词分别触发不同回复。
10. 无关输入触发“信息仍不足”的低置信度追问。
11. `继续提交 SLA` 只关闭抽屉并提示人工确认，不自动提交。
12. 原页面 `确定` 只显示模拟提交提示。
13. 全页面无真实系统接入、自动提交、最终根因或最终处理结论等越界表述。

### 2026-07-10 交互升级说明

- 顶部“知识来源配置（模拟）”模拟管理员全局策略，支持 RAG 总开关、四类来源开关和模型通用知识降级开关；刷新后从 `localStorage` 恢复。
- 页面持续展示当前可回答范围。点击智能预诊时冻结策略快照，之后修改配置不改变已经打开的对话。
- loading 只列出本次启用来源；回答同时展示已启用来源和实际命中来源。禁用来源不参与检索、排序、引用、上下文或命中统计。
- 内部资料优先；内部资料无命中时才使用已开启的开源资料；仍无命中时按通用知识降级开关进入低置信度建议或信息不足模式。
- 内置模拟截图和模拟日志可添加、预览、删除并随消息发送，不读取真实本地文件。回答提供只读诊断命令与复制按钮。
- 开源场景可将组件改为 Kafka，并在概述中输入“`KRaft metadata quorum 仲裁超时`”；命中内容和链接均为模拟数据。

本轮仍为纯前端 Demo，不调用真实接口、模型、RAG 或互联网；SLA 是否提交及建议是否适用必须人工审核。

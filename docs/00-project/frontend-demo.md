# 前端 Demo 交互说明

## 目的

本文档说明 `SLA 智能预诊助手` 第一阶段纯前端 Demo 的页面改造目标、按钮位置、点击流程、loading 效果、结果展示、mock 数据说明、交互边界和验收标准。

## 适用范围

适用于 `prototypes/sla-precheck-demo.html` 的原型评审、演示讲解和后续静态页面维护。本文只描述纯前端静态 Demo，不包含真实后端、真实 RAG、真实大模型、真实 AIOps、真实 Wiki 或真实 SLA 系统接入。

## 正文

### 页面改造目标

在模拟 AIOps SLA 报备页面中新增 `智能预诊` 按钮。用户提交 SLA 前，可以点击该按钮，基于当前表单字段模拟检索 Wiki、历史问题、历史 SLA、操作手册和产品文档，并查看参考资料、前置诊断结论和建议补充信息。

当前 Demo 只验证提交前辅助预诊的交互，不验证真实检索质量，不输出最终根因或最终处理方案。

### 按钮位置

按钮位于页面底部操作区，顺序为：

```text
取消    智能预诊    确定
```

`智能预诊` 是提交前辅助入口，不替代 `确定` 按钮，也不得阻塞原有 SLA 提交流程。

### 点击流程

1. 用户查看 Foundation 审计日志归档功能不可用的模拟 SLA 表单。
2. 用户点击 `智能预诊`。
3. 按钮进入禁用状态，文案变为 `预诊中...`。
4. 页面打开结果抽屉并展示 loading。
5. 前端使用本地 `setTimeout` 模拟 1.5 到 3 秒耗时。
6. loading 结束后展示 `智能预诊结果`。
7. 用户可关闭结果面板，或点击 `继续提交 SLA`。
8. `继续提交 SLA` 只关闭结果面板并提示人工确认，不自动提交。
9. 原页面 `确定` 仍可用于模拟提交。

### Loading 效果

loading 文案固定为：

```text
正在检索 Wiki、历史问题、历史 SLA、操作手册和产品文档，请稍候...
```

同时展示说明：当前过程使用前端 `setTimeout` 和 `模拟数据`，不调用真实接口。

### 智能预诊结果展示

结果面板标题为：

```text
智能预诊结果
```

结果内容包含：

- 检索关键词：Foundation、TDS-4.0.1、审计日志归档、功能不可用。
- 检索范围：Wiki、历史问题、历史 SLA、操作手册、产品文档。
- 前置诊断结论：当前仅基于模拟知识库返回相关资料，暂未发现可以直接解决该问题的明确方案；不代表最终根因或处理结论。
- 建议补充信息：权限、后台日志、近期变更、归档路径、存储空间和功能开关等。
- 推荐参考资料：4 条模拟资料。
- 操作提示：如果资料不能解决当前问题，请继续提交 SLA，并由研发或相关责任人进一步处理。

### Mock 数据说明

Demo 使用前端本地数组维护 mock 数据。所有条目必须标注为 `模拟数据`，链接必须使用 `example.com` 示例域。

Mock 数据范围包括 Wiki、历史问题、历史 SLA、操作手册和产品文档。禁止使用真实客户、真实 SLA 编号、真实内部链接、真实历史问题或真实系统数据。

### 交互边界

- `智能预诊` 不自动提交 SLA。
- `智能预诊` 不自动闭单。
- `智能预诊` 不判断最终根因。
- `智能预诊` 不生成最终处理方案。
- `智能预诊` 不访问真实接口。
- `继续提交 SLA` 只关闭结果面板并提示人工确认。
- 原页面 `确定` 按钮保持可用，但当前 Demo 只做模拟提交提示。

### 建议输出要求

涉及建议时必须展示：

- 依据来源：页面表单字段与前端模拟知识库数据。
- 置信度：中。
- 是否需要人工介入：需要，提交人与处理人必须确认。
- 待补充信息：真实环境日志、复现步骤、影响范围、权限与配置核查结果。

### 验收标准

1. `prototypes/sla-precheck-demo.html` 可直接在浏览器打开。
2. 页面主场景为 Foundation 审计日志归档功能不可用。
3. 底部按钮顺序为 `取消`、`智能预诊`、`确定`。
4. 点击 `智能预诊` 后按钮变为 `预诊中...` 并禁用重复点击。
5. loading 文案与本文一致。
6. loading 持续 1.5 到 3 秒。
7. 结果标题为 `智能预诊结果`。
8. 结果包含检索关键词、检索范围、前置诊断结论、建议补充信息、4 条推荐参考资料和操作提示。
9. 所有示例和链接均标注为 `模拟数据` 或使用 `example.com`。
10. 不调用任何真实接口，不影响原页面 `确定` 按钮。

# 项目进度

## 目的

本文档记录 `Transwarp Service Insight` 的当前进度、已完成事项、进行中事项、待办任务、风险边界和下一步计划。

## 适用范围

适用于项目例会、阶段评审、AI Agent 协作交接和后续迭代跟踪。当前记录仅覆盖文档、纯前端 Demo、mock 数据、架构设计和演示说明，不代表真实系统接入。

## 正文

### 总体进度

当前总体进度：第一阶段 Demo 对话式预诊改造完成，架构文档基线进度保持 40%。

当前阶段：`第一阶段：纯前端 SLA 智能预诊 Demo`，并已完成架构文档整合与目录基线。

### 已完成

(1) 明确项目定位为 `SLA 智能预诊助手`。

(2) 明确第一阶段以纯前端 Demo 为主。

(3) 明确不接入真实后端、真实 RAG、真实大模型或真实 AIOps。

(4) 完成 SLA 页面 `智能预诊` 按钮的原型设计。

(5) 完成模拟知识数据展示方案。

(6) 明确所有 Demo 数据均为 `模拟数据`，不代表真实客户、真实 SLA 或真实知识库。

(7) 明确智能预诊结果不代表最终根因或最终处理结论。

(8) 完成一期 `SLA 预诊助手` PRD 评审稿，包含 Markdown 主文档和 HTML 静态评审展示页。

(9) 在 PRD 中补充 AIOps 历史 SLA 查询和 SLA 提交能力的产品预留要求，明确不包含真实接口协议、后端实现或真实系统接入。

(10) 完成 `docs/` 分层目录重组、架构设计 docx 整合、实施计划、任务拆分、API 草案、RAG、LLM、知识库、集成、安全、观测和测试设计文档。

(11) 完成第一阶段纯前端 Demo 评审定稿，包含 Demo 入口、loading、结果面板、继续提交提示、模拟提交提示和验收说明。

(12) 完成第一阶段 Demo 对话式预诊改造，将原结果抽屉升级为聊天流，支持自动表单摘要、初步预诊回复、4 条模拟资料卡、快捷反馈、自由输入和前端静态关键词分支。

### 进行中

(1) 评审一期 `SLA 预诊助手` PRD 的试点范围、指标口径和人工审核边界。

(2) 评审架构基线、接口草案、权限脱敏、审计和降级策略。

(3) 梳理后续真实知识库检索、RAG 与大模型总结、AIOps 辅助分析的演进条件。

### 待办

(1) 基于对话式预诊 Demo 组织内部评审确认。

(2) 根据评审意见决定是否进入一期 PRD 试点或真实接入可行性评估。

(3) 评审 API 草案、知识源权限、脱敏策略、安全护栏和审计字段。

(4) 评审 RAG 原型、LLM Prompt、幻觉拦截、指标看板和 ITSM / AIOps 集成方案是否进入后续可试点范围。

(5) 准备协作者开发分支和 PR 流程。

### 风险与阻塞

- 当前 Demo 不验证真实检索效果。
- 当前 Demo 的多轮回复来自前端静态关键词分支，不代表真实模型输出。
- 当前 Demo 不包含真实权限、审计、脱敏和接口异常处理。
- 架构文档描述的是目标架构和后续设计草案，不代表当前已实现。
- 后续若接入真实知识库、AIOps 或 SLA 系统，必须单独评审并保留人工审核机制。
- 二阶段 `智能工单助手` 仅为路线方向，不能作为生产上线方案描述。

### 下一步计划

1. 使用对话式预诊 Demo 完成内部评审，并同步评审架构基线。
2. 根据评审意见调整页面交互、文案和设计草案。
3. 明确是否进入真实知识库检索可行性评估。
4. 在具备脱敏、授权和权限设计后，再讨论真实 RAG 或大模型预诊能力。

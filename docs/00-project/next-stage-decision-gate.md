# 下一阶段决策门禁

## 目的

定义第一阶段关闭后，是否进入“一期可试点系统设计评审”的决策条件，避免在范围、数据授权和人工责任未明确时直接开发 RAG、后端或真实系统集成。

## 适用范围

适用于项目负责人、产品负责人、知识库负责人、安全合规负责人、数据负责人和 SLA 业务负责人进行下一阶段决策。本文件仅为后续目标架构的评审门禁，不代表已批准试点、已启动开发或已接入任何真实系统。

## 正文

### 当前决策

当前决策为：**暂不进入真实 RAG、LLM、AIOps 或后端实施，仅准备一期可试点系统设计评审材料**。

进入设计评审前，必须同时满足 Demo 验收关闭、试点范围明确、指标基线获批、数据授权可证明、权限与审计责任清晰等条件。

### 人工审核 RACI

角色含义：`R` 为执行责任，`A` 为最终负责，`C` 为协商，`I` 为知会。同一事项只能有一个 `A`；姓名和代理人由业务评审补充。

| 人工审核事项 | 提交人 | 处理人 | 知识维护人员 | 运营负责人 | 必须保留的人工结果 |
| --- | --- | --- | --- | --- | --- |
| 是否提交 SLA、提交内容准确性 | R/A | C | I | I | 提交人确认记录 |
| 问题分类和影响范围 | C | R/A | I | C | 处理人复核记录 |
| 处理建议是否适用于当前环境 | C | R/A | C | I | 处理人确认或退回意见 |
| 复盘报告完整性 | R | R/A | C | I | 提交人和处理人共同确认 |
| 知识条目准入、更新和撤回 | C | C | R | A | 知识审核与版本记录 |
| 指标口径、基线和试点运营 | I | C | C | R/A | 口径审批与基线签字记录 |

智能预诊只提供辅助建议，不承担上述任何 `A` 责任；SLA 不自动提交、不自动闭单，知识条目未经提交人和处理人确认不得进入沉淀流程。

### 进入设计评审的硬门禁

以下条件必须全部满足：

1. 第一阶段 Demo 的 Edge 人工冒烟、业务口径确认和项目负责人签字均已关闭。
2. 试点用户、组件、问题类型和排除范围已获批。
3. 知识来源清单及权限、脱敏、更新、撤回和审计责任已获批。
4. 四项核心指标的当前基线已计算并由业务负责人批准，试点目标不以未经验证的降单数量为强压指标。
5. 人工审核责任、失败降级路径和“不自动提交、不自动闭单”边界已书面确认。
6. AIOps 能力明确为仅预留，或通过单独评审后才进入设计范围。

只有硬门禁全部通过后，才可评审 API 草案、知识权限、安全护栏、审计字段、降级策略以及后续 RAG/LLM 可行性；通过设计评审仍不等于批准生产开发或真实接入。

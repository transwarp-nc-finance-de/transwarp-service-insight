# 架构演进计划

## 目的

本文档说明从当前 Demo 到目标架构的演进路线，避免将后续能力误描述为当前已实现能力。

## 适用范围

适用于架构评审、路线规划和阶段验收。所有后续能力均需单独评审。

## 正文

1. 文档基线阶段：完成目录重组、架构设计、任务拆分和边界说明。
2. Demo 增强阶段：优化纯前端交互和 mock 数据，继续标注 `模拟数据`。
3. 可试点系统设计阶段：评审 API、权限、审计、知识源、脱敏和集成字段。
4. RAG 原型阶段：基于授权脱敏数据验证检索、重排序和评估。
5. LLM 预诊阶段：验证结构化输出、引用对齐、幻觉拦截和人工审核。
6. 集成闭环阶段：在人工确认前提下探索 ITSM / AIOps 集成和指标闭环。

每个阶段都必须保留人工审核，不得自动提交 SLA、自动闭单、自动判责或自动修改生产配置。

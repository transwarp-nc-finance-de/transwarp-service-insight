# 部署规划

## 目的

本文档记录目标架构的部署规划和当前阶段边界，便于后续基础设施评审。

## 适用范围

适用于后续可试点系统、RAG 原型、LLM 预诊和集成部署评审。当前阶段不创建生产部署脚本，不创建 K8s、Helm 或后端服务目录。

## 正文

目标架构可按以下部署域规划：

- 前端域：工单页嵌入面板或独立静态资源。
- 业务服务域：预诊编排、反馈记录、权限审计。
- 检索域：ES、向量库、重排序服务和缓存。
- LLM 域：模型推理、模型路由、熔断降级。
- 数据域：知识元数据、审计日志、埋点事件和指标数据。
- 观测域：日志、指标、告警和看板。

当前仓库仅保留设计文档和纯前端 Demo，不部署真实服务，不连接真实数据库，不申请真实 GPU 或生产资源。

# Mock 知识数据说明

## 目的

本文档说明第一阶段纯前端 Demo 使用的 mock 知识数据和对话回复规则，确保所有示例数据均保持演示边界，不被误解为真实知识库命中或真实模型输出。

## 适用范围

适用于 `prototypes/sla-precheck-demo.html` 中的前端 mock 数据、对话式预诊回复、关键词分支维护，以及 Demo 讲解、评审和后续样例扩展。

## 正文

### Mock 数据目的

Mock 数据用于模拟 `智能预诊` 点击后的对话式参考资料展示效果，帮助评审者理解 SLA 提交前如何展示资料来源、辅助结论、置信度、人工介入建议和待补充信息。

Mock 数据不用于证明真实知识库质量，不代表真实 Wiki、真实 SLA、真实历史问题、真实客户现场或真实大模型输出。

### Mock 数据来源说明

当前所有 mock 数据均由项目文档人工构造，统一标注为 `模拟数据`。数据内容不来自真实客户、真实历史 SLA、真实 Wiki、真实内部系统或真实 AIOps 平台。

### 前端静态数组

Demo 使用前端本地静态数组维护参考资料，不调用真实接口。

```js
const mockKnowledgeResults = [
  {
    type: "Wiki",
    title: "Foundation 审计日志归档功能说明",
    summary: "模拟数据：介绍 Foundation 审计日志归档相关配置、使用限制和常见问题。",
    url: "https://example.com/wiki/foundation-audit-archive"
  }
]
```

### Mock 数据字段说明

- `type`：资料类型，例如 Wiki、历史问题、历史 SLA、产品文档。
- `title`：资料标题，必须为模拟标题，不得使用真实内部标题。
- `summary`：资料摘要，必须以 `模拟数据` 标注。
- `url`：示例链接，必须使用 `example.com` 示例域，不得使用真实内部链接。

### 对话回复规则

对话式预诊回复同样为前端静态规则生成，不代表真实大模型输出。当前分支包括：

- 初始回复：基于表单字段和前端 mock 资料生成，置信度为中。
- 日志/报错分支：提示补充日志时间点、组件日志路径、完整报错上下文和复现步骤。
- 权限分支：提示核查审计管理权限、角色配置和功能开关。
- 影响范围分支：提示确认全员、单用户或单集群影响，并由人工判断 SLA 级别。
- 近期变更分支：提示核查升级、配置调整、变更记录和配置差异。
- 未命中分支：提示信息不足，置信度为低，要求继续补充日志、权限、影响范围或近期变更。

所有回复必须包含：

- `模拟数据` 或等效边界说明。
- 依据来源。
- 置信度。
- 人工介入建议。
- 待补充信息。

### 使用限制

- 只能用于纯前端 Demo。
- 不得被描述为真实命中结果。
- 不得被描述为真实模型输出。
- 不得包含真实客户名称。
- 不得包含真实 SLA 编号。
- 不得包含真实日志片段。
- 不得包含真实内部系统地址。
- 不得调用真实 Wiki、AIOps、SLA 或数据库接口。

### 脱敏要求

如后续需要基于真实材料制作演示数据，必须先完成脱敏和授权。脱敏至少包括：替换客户名称、项目名称和集群名称，移除真实 SLA 编号、工单编号、内部链接、IP、账号、路径、密钥、日志详情和业务敏感字段，并用明确的 `模拟数据` 标识替换真实来源。

### 建议输出要求

当 mock 数据用于生成建议时，必须说明：

- 依据来源：页面表单字段、前端模拟知识库数据和用户补充的模拟上下文。
- 置信度：中或低。
- 是否需要人工介入：需要。
- 待补充信息：真实环境日志、复现步骤、影响范围、权限与配置核查结果、近期变更记录。

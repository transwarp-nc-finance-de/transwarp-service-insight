# 安全与护栏设计

## 目的

本文档定义 SLA 智能预诊后续目标架构中的安全、合规、幻觉拦截和人工审核护栏。

## 适用范围

适用于安全评审、LLM 评审、RAG 评审和集成评审。当前阶段不接真实敏感数据。

## 正文

安全设计包括：数据不出域、权限过滤、脱敏、传输与存储加密、Prompt 注入防御、指令与用户输入隔离、生成前拦截、生成中约束、生成后引用对齐、实体一致性检查、敏感信息过滤、高风险结果降级、审计日志和人工审核。

模型输出只作为辅助建议，不得作为最终根因、最终处理结论或正式复盘结论。系统不得绕过人工审核，不得输出未脱敏敏感信息，不得根据用户注入指令改变系统规则。

### 来源准入、附件与命令护栏增补

- 管理员策略由后续服务端产生并签入策略版本；普通用户请求不能开启被管理员关闭的来源。
- 权限、`APPROVED` 状态和敏感级别过滤必须发生在召回与模型上下文构造之前；禁用来源不得进入日志中的正文、引用或缓存上下文。
- 开源资料只允许官方文档、发布说明、安全公告和上游 Issue/PR，并保留域名准入、内容安全和许可证评审。
- 附件必须在上传前提示脱敏，审计只记录附件类型、数量、脱敏状态和摘要标识，不记录未脱敏正文。当前 Demo 不读取真实附件。
- 命令输出采用只读白名单和风险分类；删除、修改、重启、扩权、绕过审核等操作不得生成。
- 策略变更只影响新预诊，历史对话保留当时策略快照，便于追溯且避免引用口径漂移。

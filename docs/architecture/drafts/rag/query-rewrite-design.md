# 查询改写设计

## 目的

本文档描述后续目标架构中的查询改写、意图识别和实体规范化设计。

## 适用范围

适用于 RAG 检索质量优化和审计设计。当前 Demo 不执行真实查询改写。

## 正文

查询改写目标是将用户口语化、碎片化输入转换为更适合检索的查询，同时保留原始输入用于审计。

设计能力包括：产品术语纠错、产品名规范化、错误码识别、同义词扩展、IP / 日期 / 版本实体识别、意图分类和检索过滤条件推导。

风险是过度改写导致误召回，因此后续实现应同时保存 `original_query` 与 `rewritten_query`，并在预诊结果中说明改写依据。任何改写都不能改变用户原始问题含义。

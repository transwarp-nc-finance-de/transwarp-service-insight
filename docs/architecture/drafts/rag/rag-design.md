# RAG 设计

## 目的

本文档描述后续目标架构中的 RAG 能力设计，用于支撑 SLA 智能预诊的可追溯辅助建议。

## 适用范围

适用于 RAG 原型评审、检索服务设计、知识库设计和 LLM 生成设计。当前 Demo 不具备真实 RAG 能力。

## 正文

目标 RAG 流程包括：查询改写、意图识别、实体抽取、文档解析、文档切片、元数据标注、Embedding、关键词检索、向量检索、多路召回、去重合并、重排序、L1 / L2 缓存、缓存失效和检索质量评估。

真实数据必须满足授权、脱敏、可审计和已审核要求。只有完成索引并处于 `PUBLISHED` 的当前知识版本可进入检索；`APPROVED` 但未发布的版本不得参与。检索结果必须展示来源、适用范围、更新时间或版本信息，不能被描述为最终根因。

评估指标包括 Top1 / Top3 / Top5 命中率、MRR、NDCG、采纳率、低分反馈和高频未命中 query 分析。

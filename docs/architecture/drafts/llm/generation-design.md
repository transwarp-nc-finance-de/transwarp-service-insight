# 生成设计

## 目的

本文档描述后续目标架构中 LLM 生成管理、模型路由、结构化输出和降级策略。

## 适用范围

适用于 LLM 预诊能力评审。当前仓库不接真实模型，不创建推理服务。

## 正文

生成管理器负责组织 Prompt 模板、检索上下文、模型调用、结构化输出解析和降级处理。简单问题可路由到轻量模型，复杂诊断可路由到更强模型，具体模型选择需后续评审。

输出必须是可解析结构，解析失败时降级为“仅展示检索结果和待补充信息”。推理超时、模型错误或安全拦截时，也必须允许用户继续人工提交 SLA。

生成内容必须可追溯到引用资料，不得凭空补充不存在的处理结论。

### 回答模式与上下文约束增补

后续生成管理器应接收服务端计算的策略快照和有效来源，不接受普通用户自行扩大来源范围。输出 `answer_mode` 取值为内部资料、开源资料、通用知识或信息不足。

- 内部资料/开源资料模式：只将本次实际命中的引用片段放入模型上下文。
- 通用知识模式：不伪造资料引用，固定显示低置信度和降级原因。
- 信息不足模式：不生成排查结论，只返回待补充信息和人工继续提交 SLA 的入口。
- 所有模式：展示配置来源、实际命中来源、策略版本、依据、置信度、人工介入建议、待补充信息及非最终结论声明。
- 命令安全：仅允许白名单中的只读查看、查询、状态检查命令；禁止删除数据、修改配置、重启服务、变更权限或执行生产操作。

当前 Demo 使用静态模板模拟结构化回答和复制按钮，不调用真实模型。

# 知识生命周期

## 目的

本文档定义后续目标架构中的知识接入、处理、审核、发布、过期和审计流程。

## 适用范围

适用于知识库建设、RAG 检索和知识审核流程评审。当前 Demo 仅使用前端 mock 数据，不接真实知识库。

## 正文

目标生命周期包括多源接入、文档解析、切片、元数据标注、向量化、审核、发布、过期、废弃和审计。

状态机：

```text
DRAFT -> REVIEW -> APPROVED / REJECTED -> EXPIRED / DEPRECATED
```

只有 `APPROVED` 知识可被线上检索。`DEPRECATED` 仅用于审计和历史追溯。所有状态变更必须记录操作人、时间、原因和审核意见。

真实知识必须经过授权、脱敏、权限过滤和人工审核后才能进入检索范围。

# 离线知识处理流程

Status: DRAFT  
Owner: 知识治理负责人  
Last reviewed: 2026-07-12  
Source of truth for: 未来知识导入、审核和发布边界

```text
白名单数据源 → 导入 → 解析 → 元数据/权限 → 人工审核
→ APPROVED → 发布版本 → 建索引 → 可检索
                       ↘ 失败状态、重试与重建
```

只有授权、脱敏、审核通过且在有效期内的版本可进入线上检索。导入、解析、切片、审核、发布、废弃和重建索引必须有独立状态与版本，不得在在线预诊请求中执行。

当前仓库只允许使用模拟领域对象和内存测试 Adapter，不读取真实文件、不连接真实知识源、不构建真实索引。数据源授权、版权、保留删除策略、权限继承和外部存储均需人工确认。

## 一期已确认目标边界

- 导入格式为 Markdown、TXT 与包含可直接提取文本层的 PDF。
- 扫描 PDF 与图片 OCR 不属于一期；DOCX、HTML 和 Wiki 自动同步也不属于一期。
- 文本型 PDF 的段落正常提取，表格按阅读顺序尽力展平，图片忽略。解析结果携带内容丢失或顺序不确定警告，并在预览中展示；该警告不等于解析失败，但未获审核人员确认的版本不得发布。
- 原始文件不可变。编辑人员可修正元数据、适用范围、清洗文本和解析警告说明；每次修改产生新草稿修订并使既有 Chunk/索引结果失效。审核中修改退回 `DRAFT`，重新生成解析/切片预览后方可再次提交审核。
- `KnowledgeVersion` 只表达 `DRAFT → IN_REVIEW → APPROVED → PUBLISHED → DEPRECATED` 的人工治理生命周期；`ParseTask` 独立表达 `PENDING → RUNNING → SUCCEEDED/FAILED` 的技术处理生命周期，并通过版本 ID 关联。解析失败可重试，不改变知识版本的审批结论。
- `PUBLISHED` 版本不可原地修改。修改创建新 `DRAFT`；新版本发布与旧版本转为 `DEPRECATED` 必须原子完成。同一文档最多一个版本可检索，历史版本仍保留其稳定标识。
- `APPROVED` 版本先通过独立 `IndexTask` 构建 FTS 与向量索引，两路都成功后才原子激活为 `PUBLISHED` 并废弃旧版本。失败时保持 `APPROVED`，旧发布版本继续服务；半索引版本不得进入检索。
- `IndexTask` 对数据库暂时不可用、Embedding 超时等瞬时错误退避重试，总 attempt 最多 3 次；维度不匹配、内容无效等确定性错误不自动重试。耗尽后由授权人员在修复原因后创建新任务，历史失败任务不可变。
- 解析器保持可替换边界，后续格式只有在具备授权样本和质量验收标准后才扩展。
- 单文件上限为 Markdown/TXT 5 MB、文本型 PDF 20 MB，按原始字节数在写入持久存储前校验；超限不创建可处理的解析任务，管理员也不得绕过。
- 瞬时 I/O 或临时资源错误采用退避重试，总 attempt 最多 3 次；确定性格式、大小或文本层错误不自动重试。次数耗尽进入 `FAILED`，修正输入后显式创建新任务，旧失败任务保持不可变。
- 切片采用结构优先的确定性策略，优先保留标题、段落和 PDF 文本块边界；超长结构块使用最大 400 Token、重叠 50 Token 的窗口。规则与窗口参数版本化并进入索引/审计元数据，不提供 UI 调参。重新切片创建新的索引任务，不改写既有 Run 引用的 Chunk。

## 当前实施状态

Issue #21 已实现首次上传、不可变 Compose volume 文件、Markdown/TXT/文本型 PDF 解析、最大三次 ParseTask、解析摘要以及 Block/Chunk 分页预览。扫描 PDF、OCR、草稿修订、审核、发布、FTS、pgvector 和 Evidence 仍未实现；当前切片不代表一期完成。

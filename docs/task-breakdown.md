# 任务拆解

## 目的

本文档按阶段拆解 `Transwarp Service Insight` 的工作任务，明确每项任务的目标、内容、输入、输出、验收标准、状态和预计进度百分比。

## 适用范围

适用于项目管理、进度跟踪、任务分派和后续迭代规划。当前进度为总项目命名与一阶段 `SLA 预诊助手` 路线确认阶段。

## 阶段 0：总项目命名与一阶段路线确认

- 目标：建立统一项目总名、业务定位、阶段路线和文档结构。
- 内容：统一 `Transwarp Service Insight` 总项目命名，明确一阶段 `SLA 预诊助手`，保留人工审核和 Demo 边界。
- 输入：项目计划、业务定位和阶段路线。
- 输出：规范化 Markdown 文档。
- 验收标准：文档完整、边界清晰、无真实客户数据、无代码实现、无真实系统接入声明。
- 状态：进行中。
- 预计进度：8%。

## 阶段 1：SLA 预诊业务流程和字段标准确认

- 目标：明确 SLA 提交前预诊和复盘所需字段。
- 内容：梳理提交字段、处理字段、审核字段、知识沉淀字段，以及基础类、重复类、开源社区类问题识别字段。
- 输入：现有 SLA 流程、用户访谈、流程规范。
- 输出：字段清单和字段说明。
- 验收标准：提交人、处理人和审核人均可理解字段含义，且能支撑提交前预诊。
- 状态：待办。
- 预计进度：15%。

## 阶段 2：模拟知识库和 Demo 数据准备

- 目标：准备可演示的模拟售后知识库、模拟 SLA 复盘数据、模拟 Wiki、模拟内部培训素材、模拟使用手册、模拟历史 SLA 和模拟 AIOps 指标。
- 内容：设计模拟数据结构，编写样例，标注来源和适用范围。
- 输入：Demo 场景、知识库模板、指标模板、复盘模板。
- 输出：模拟数据集和演示脚本。
- 验收标准：所有数据均标注为模拟数据，不含真实客户、真实知识库内容或真实接口。
- 状态：待办。
- 预计进度：25%。

## 阶段 3：预诊与检索流程设计

- 目标：形成端到端 SLA 提交前预诊流程。
- 内容：定义输入、补充问题、分类建议、检索摘要、指标摘要和是否建议提交 SLA 草稿的展示方式。
- 输入：字段清单、模拟知识库、模拟 AIOps 数据、模拟 SLA 复盘数据。
- 输出：预诊流程说明和交互样例。
- 验收标准：能够覆盖信息补全、相似案例、指标摘要、提交建议和人工审核。
- 状态：待办。
- 预计进度：40%。

## 阶段 4：SLA 草稿与复盘模板完善

- 目标：形成可人工审核的 SLA 草稿和复盘报告模板。
- 内容：完善标题、描述、影响范围、优先级、时间线、根因、改进建议和知识沉淀字段。
- 输入：Demo 场景、处理记录样例、模板文档。
- 输出：SLA 草稿模板和复盘报告模板。
- 验收标准：模板能区分辅助建议、待确认字段和人工最终结论。
- 状态：待办。
- 预计进度：55%。

## 阶段 5：审核与治理机制设计

- 目标：明确人工审核、权限、日志和审计要求。
- 内容：定义审核节点、角色权限、日志追踪、模型输出留痕和知识发布审批。
- 输入：业务流程、权限需求、审计要求。
- 输出：审核流程和治理说明。
- 验收标准：关键动作均有人工确认和记录。
- 状态：待办。
- 预计进度：70%。

## 阶段 6：二阶段方向评审与试点规划

- 目标：在一阶段评审后，判断是否进入二阶段智能工单助手方向或一阶段试点。
- 内容：定义试点范围、评估指标、风险控制和迭代计划；仅记录二阶段方向，不直接扩展为生产上线方案。
- 输入：Demo 反馈、流程评审意见、数据准备情况。
- 输出：试点计划、二阶段方向评审结论和迭代路线图。
- 验收标准：试点不依赖未授权真实数据，不绕过人工审核，不宣称真实客户或合作伙伴侧接入。
- 状态：待办。
- 预计进度：100%。

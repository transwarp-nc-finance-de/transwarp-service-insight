# AI Agent 协作规范

## 目的

约束 `Transwarp Service Insight` 技术 MVP 开发，确保项目围绕“借助 AI 能力提升交付与服务效率”推进，并聚焦 `SLA 智能预诊助手`。

## 适用范围

适用于本仓库的文档、纯前端 Demo、Spring Boot 后端、Vue 前端、Mock API、测试以及后续技术原型工作。

## 当前阶段

第一阶段纯前端 Demo 已于 2026/07/10 关闭，当前进入技术 MVP 开发。允许实现 Spring Boot、Vue、Mock API、自动化测试，以及在后续里程碑中评估数据库、RAG 和 Compose 原型。当前范围以 `docs/CURRENT_SCOPE.md` 为唯一短版入口。

## 工作原则

- 使用中文输出，保持范围收敛；代码命名遵循技术栈惯例。
- 仅使用模拟、公开或已脱敏数据；模拟内容必须标注 `模拟数据`。
- 智能输出必须包含依据来源、置信度、人工介入建议和待补充信息。
- 建议不得阻断人工继续提交，不得写成最终根因、最终处理结论或正式复盘结论。
- SLA 提交、内容准确性、分类、影响范围、建议适用性、复盘和知识沉淀均由人工确认。

## 当前允许事项

- 修改文档、Mock 数据、静态 Demo、页面样式和交互。
- 开发 `backend/` Spring Boot 服务、`frontend/` Vue 应用、Mock API 与测试代码。
- 编写 API、RAG、LLM、知识库、集成、安全、观测和验收设计草案。
- 后续按 Backlog 开展数据库、真实 RAG 前置验证和 Compose 原型，但不得宣称已真实接入。

## 硬边界

- 不使用未脱敏的真实客户、日志、SLA、内部链接或敏感附件。
- 不虚构真实客户、故障、接口地址、数据库连接或外部平台接入状态。
- 不自动提交或关闭 SLA，不自动判责，不自动修改生产配置，不绕过人工审核。
- 不把模型或 Mock 输出作为最终根因、最终方案或知识库正式条目。
- 未经明确授权，不接入真实 ITSM、AIOps、监控、Wiki、历史 SLA、RAG、模型、数据库或生产环境。

## 变更与确认

- 普通开发提示词无需归档；仅对需要长期复用或具有治理意义的重要提示词归档到 `prompts/`。
- CHANGELOG 在形成可演示里程碑或重大范围变化时更新，不要求逐次文件修改都记录。
- 仅当涉及真实数据、外部服务、破坏性操作、已发布公共接口兼容性或重大业务歧义时暂停并请求人工确认。
- 优先修改当前任务直接相关文件，保留人工审核约束，并显式记录新增假设、风险和限制。

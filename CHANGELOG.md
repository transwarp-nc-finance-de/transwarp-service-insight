# 变更记录

## 目的

本文档用于记录 `Transwarp Service Insight` 的重要需求边界、文档规则、目录结构和 Demo 原型变更，便于后续追溯。

## 适用范围

适用于本仓库内由 AI Agent 或人工协作完成的重要项目修改。当前阶段仅记录文档、纯前端 Demo、mock 数据和规则层面的变更，不代表真实系统接入、生产上线或客户环境变更。

## 记录规则

1. 按日期倒序记录，最新日期放在最上方。
2. 日期格式统一为 `YYYY/MM/DD`，例如 `2026/07/03`。
3. 同一天的多条变更必须合并到同一个日期标题下。
4. 不得重复创建相同日期标题。
5. 每条变更使用编号列表：`(1)`、`(2)`、`(3)`。
6. 每条变更尽量说明“改了什么”和“为什么改”。
7. 只记录有意义的项目变更，不记录临时调试、无关格式调整或个人操作。
8. 涉及真实客户、真实 SLA、真实日志或敏感信息时，必须先脱敏再记录。
9. 如果只是纯前端 Demo 或模拟数据变更，需要明确标注 `Demo` 或 `模拟数据`。

## 变更记录

### 2026/07/03

(1) 新增 Prompt 管理规则，明确重要 AI Agent 提示词需归档到根目录 `prompts/`，用于保证后续提示词来源可追溯。

(2) 规范提示词文件命名要求，采用 `YYYY-MM-DD-序号-主题.md` 格式，同日编号使用 `01`、`02`、`03`，文件名主题部分使用英文小写和短横线，避免后续归档混乱。

(3) 明确提示词内容应包含使用时间、使用目的、适用范围和提示词正文，并要求使用提示词完成项目修改后同步更新 `CHANGELOG.md`。

(4) 重构项目目录结构，将前端 Demo 原型迁移到 `prototypes/sla-precheck-demo.html`，并将 `docs/` 收敛为 Markdown 文档目录，避免原型文件和项目文档混放。

(5) 统一功能文案，将当前对外能力收敛为 `SLA 智能预诊助手`，将按钮、结果标题和当前阶段描述统一为 `智能预诊`、`智能预诊结果` 和 `第一阶段：纯前端 SLA 智能预诊 Demo`，避免把底层知识检索动作误写成产品能力。

(6) 重写纯前端 Demo，使用 `Foundation 审计日志归档功能不可用` 作为主场景，并通过本地 `setTimeout` 和前端 mock 数据模拟 Wiki、历史问题、历史 SLA、操作手册和产品文档检索；所有示例均标注为 `模拟数据`，不调用真实接口。

(7) 更新 `AGENTS.md`，新增当前阶段允许事项、当前阶段禁止事项和 `CHANGELOG` 更新规则，明确当前阶段只允许文档、纯前端静态 Demo、mock 数据和演示说明，不允许后端、真实 RAG、真实大模型、真实 AIOps 或真实 SLA 系统接入。

(8) 重写 `README.md` 和 `docs/` 下核心文档，将旧的泛化 RAG、AIOps 和智能复盘方案收敛为第一阶段 Demo 范围，并把真实知识库检索、RAG 与大模型总结、AIOps 辅助分析、SLA 表单智能补全和智能复盘报告生成移入后续演进方向。

(9) 完善 `.gitignore`，补充 OS、IDE、日志、依赖、构建产物、环境变量、临时文件和误包含本地目录规则，用于避免本地缓存、敏感配置和构建产物进入仓库。

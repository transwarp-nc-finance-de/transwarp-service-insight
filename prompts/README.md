# Prompt 归档规则

## 目的

本文档用于规范 `Transwarp Service Insight` 项目中重要 AI Agent 提示词的归档方式，确保后续文档修改、需求整理、Demo 设计和方案评审过程可追溯。

## 适用范围

适用于本仓库根目录 `prompts/` 下保存的重要提示词文件。普通临时对话不强制归档；用于推动项目文档修改、需求调整、方案重构或关键评审的提示词应归档。

## 正文

### 存放位置

重要提示词必须保存到根目录 `prompts/`。

### 命名规则

提示词文件名格式为：

`YYYY-MM-DD-序号-主题.md`

示例：

`2026-07-03-01-sla-precheck-demo-restructure.md`

命名要求如下：

- 日期使用提示词实际使用日期。
- 同一天多个提示词使用 `01`、`02`、`03` 编号。
- 主题部分使用英文小写和短横线。
- 文件扩展名使用 `.md`。

### 内容要求

每个提示词文件应至少包含以下内容：

- 使用时间。
- 使用目的。
- 适用范围。
- 提示词正文。

### 变更记录要求

使用提示词完成项目修改后，必须同步更新根目录 `CHANGELOG.md`，记录本次变更日期、变更内容和影响范围。

### 约束说明

提示词归档不得突破 `AGENTS.md` 中定义的项目定位、禁止事项、人工审核要求和变更管理要求。涉及 Demo 的内容必须明确标注为“模拟数据”，不得描述为真实系统接入或真实客户案例。
